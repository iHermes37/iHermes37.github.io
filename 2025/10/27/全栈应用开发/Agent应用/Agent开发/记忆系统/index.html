记忆系统

在 OpenAI 应用研究主管 LilianWeng 写的万字长文中：LLM Powered Autonomous Agents | Lil’Log，她提出：

Agent = LLM+ 记忆 + 规划技能 + 工具使用。

在之前的文章里我们简单聊过了：MCP协议(为agent赋能工具使用) 和 提示词工程(为任务进行规划)，而在本文中则重点聊一聊agent的记忆系统。

概述

类比于人类，记忆是指对信息的获取、存储、保持与后续检索过程。认知心理学通常将人类记忆划分为多种类型：

  • 感觉记忆（Sensory Memory）：对视觉、听觉等原始刺激的极短暂保留，持续时间通常只有几百毫秒到数秒，用于决定哪些信息进入注意阶段。

  • 短期记忆 / 工作记忆（Short-term / Working Memory）：保存当前意识中的信息，用于推理、决策与问题求解，容量有限（经典结论约为 7±2 个项目），持续时间约 20–30 秒。

  • 长期记忆（Long-term Memory）：可长期保存信息，容量巨大，进一步分为：

    • 显性 / 陈述记忆：包括情景记忆（个人经历）与语义记忆（事实与概念）；
    • 隐性 / 程序记忆：以技能和习惯的形式存在，如骑车、打字等自动化行为。

    进一步思考可以发现：隐性/程序记忆体现为“无需思考即可执行的技能”。在 Agent 系统中，其对应形式通常不是显式存储的数据，而是：

    • 固化在 Prompt 模板中的行为模式（如 ReAct、Plan-and-Execute 结构）
    • 编排在 Agent 工作流图（LangGraph / AutoGen Flow）中的执行策略

    以及模型本身在预训练阶段学到的隐含推理路径

虽然 AI Agent 并不存在生物学意义上的记忆结构，但在工程实现上，如果希望 Agent 具备连续对话能力、跨任务知识积累能力以及复杂规划能力，就必须构建一套在功能上对应上述记忆分层的存储与检索机制。因此，当前主流 Agent 框架通常将“记忆系统”拆解为如下三类：

记忆类型 描述 类比人类记忆 技术与实现
短期记忆（Short-term Memory） 保存最近的原始对话流与中间推理结果，是上下文的“滑动窗口缓冲区”。 对刚刚发生事件的即时记忆，对应人类的短期/工作记忆的瞬时部分。 通常就是 Prompt 中的历史对话窗口；实现上是固定长度的列表或 Token 截断策略，存储 (用户输入, Agent响应, 中间推理)。
长期记忆（Long-term Memory） 从多轮交互与多任务中抽取出的稳定知识与经验，用于跨会话复用。 对应人类的长期语义记忆与情景记忆。 核心是向量数据库 + Embedding 模型；将文本、事件、总结转成向量，通过语义相似度检索相关记忆。
工作记忆（Task / Working Memory） 为当前任务临时组装的“认知工作台”，整合所有与当前目标最相关的信息。 人类在解决问题时主动回忆并集中注意的信息集合。 一个临时数据结构，在任务开始时由调度器构建：= 当前指令 + 短期上下文 + 从长期记忆检索到的相关片段 + 工具返回结果。

从系统架构角度看：

  • 短期记忆 = 上下文管理问题
  • 长期记忆 = 信息检索问题（IR / RAG / 向量索引）
  • 工作记忆 = 任务编排与状态管理问题

这三者共同决定了 Agent 是否具备：

  • 连续对话能力
  • 跨任务学习能力
  • 多步规划与工具协同能力

结合上图，一个典型的工作流程如下：

  1. 记忆写入：
    • 原始记录：每一次与用户的交互都会被自动存入短期记忆。
    • 总结提炼：在对话间隙或任务结束时，系统会主动总结对话要点，并将关键信息（如用户偏好、重要结论）转化为向量，存入长期记忆（向量数据库）。
  2. 记忆读取（检索）：
    • 触发：当新的用户查询到来时，记忆系统被激活。
    • 多路检索：系统会同时进行：从短期记忆中获取最近的几条对话，保持话题连贯。将用户当前查询向量化，在长期记忆（向量数据库/关系数据库）中进行语义搜索，找到历史上相关的对话片段和知识。
  3. 记忆合成：所有检索到的信息（近期对话 + 相关长期记忆）被共同组装到工作记忆中，形成一份为当前任务定制的、完整的上下文背景。
  4. 决策与行动：这份丰富的工作记忆被作为上下文，与当前的用户问题和系统指令一同提交给大语言模型。LLM 基于这份“带有记忆的上下文”做出更精准、更个性化的决策和回答。
