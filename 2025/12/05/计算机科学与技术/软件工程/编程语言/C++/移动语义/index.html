特性 Rust 所有权转移 C++ 移动语义
设计目标 内存安全，无GC 性能优化，避免拷贝
强制程度 编译时强制，违反则编译失败 运行时行为，可被忽略或误用
默认行为 转移（除非显式复制） 拷贝（除非显式移动）
安全性 100%安全，无悬垂指针 可能出错，需开发者保证
生命周期 编译器跟踪，静态检查 开发者负责，动态风险

总结：C++移动语义的本质

  1. **std::move 只是”建议移动”**，不是”执行移动”
  2. **移动是”资源管理权的转移”**，不是”魔法”
  3. 编译器不保证安全，需要开发者自己保证
  4. 移动后的对象状态不确定，取决于实现
  5. 对比Rust：C++移动是”建议性、手动的”，Rust是”强制性、自动的”

您的比喻非常准确：

  • C++移动：像借车给朋友，你还能开（但不应该），朋友可能还（也可能不还）
  • Rust移动：像卖车过户，车不再是你的，你不能再开
