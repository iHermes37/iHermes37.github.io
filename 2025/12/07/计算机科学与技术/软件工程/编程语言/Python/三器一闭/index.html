三器一闭

闭包

在 Python 中，函数是第一类对象，可以作为返回值。但如果一个函数返回后，它的局部变量按理会被销毁，如何让内部函数继续访问这些变量？闭包就是解决这个问题的机制。

闭包是一个函数与其引用环境（外部变量）的组合。简单说：内部函数记住了外部函数作用域的变量。

闭包 = 函数 + 外部作用域的局部变量（被捕获并持续存在）
即：内部函数“记住了”外部函数当时的变量环境，即使外部函数已经返回。

闭包三要素

  1. 嵌套函数：函数内部定义函数
  2. 内部函数引用外部变量
  3. 外部函数返回内部函数

重要结论：装饰器就是闭包的应用！装饰器 decorator 本质上就是 使用闭包的语法糖。

闭包是 Python 函数式编程的核心能力： 让函数成为携带状态的可执行对象。

生成器

生成器是一种特殊的可迭代对象，它的执行是“暂停式”的。

一句话：生成器 = 带有 yield 的函数，它在执行过程中可以随时暂停，并在下一次迭代时继续执行。

和普通函数的最大区别：

  • 普通函数：一次性执行到底 → 返回结果 → 结束
  • 生成器函数：执行到 yield → 暂停 → 返回一个值 → 下次继续从暂停点执行
特性 生成器 普通函数
返回值 yield 多次返回值 return 一次返回值
状态 保持状态，可暂停恢复 不保持状态
内存 按需生成，内存高效 可能占用大量内存
使用场景 大数据流、无限序列、管道处理 确定计算、需要立即结果

因此生成器具有惰性（lazy）计算的能力。

生成器主要解决两个核心问题：

  • 节省内存（按需生成）：例如要处理 10 亿条数据：
    • 列表：一次性创建，占用巨大内存
    • 生成器：每次只产生一个元素，内存消耗近乎 0
  • 实现“流式处理”：数据可以像流水一样一边生成、一边消费，不需要提前构建完整结构。
  • 可以做协程的基础：在 async/await 出现之前，Python 的协程就是生成器扩展语法。

生成器本质上是一个 状态机，内部维护：

  • 当前执行位置
  • 局部变量
  • 执行栈

迭代器

迭代器解决的是数据太大的问题。例如：读取 100GB 日志

  • list：会把所有行读入 → 崩溃
  • 迭代器：一次读一行 → 内存稳定在几 KB

因此：迭代器让“延迟计算”“流式处理”在 Python 里成为第一等公民。

迭代器是一类能逐个产出元素的对象，它实现了两个方法：

  • __iter__()：返回自身（迭代器本身）
  • __next__()：返回下一个值；如果没有更多值，抛出 StopIteration

换句话说：迭代器是一个“可被逐步取值”的对象，它不会一次性把数据加载全部内存，而是按需取值。这是 Python 整个“可迭代对象生态”的核心。

因为生成器对象自动实现了 __iter__() 和 __next__()，生成器对象 天然就是迭代器。

可迭代对象：list、dict、str、tuple、set…它们本身不能“自己”按步骤往前走for 循环会自动把它们转成迭代器

迭代器：

  • 文件对象
  • 生成器
  • map/filter 对象
  • iter() 得到的对象

当写：

Python 做了 3 件事：

  1. 调用 iter(data)
    • 若是可迭代对象 → 返回一个迭代器
    • 若是迭代器 → 返回它自己
  2. 不断调用 next(iterator) 获取下一个元素
  3. 遇到 StopIteration → 结束循环
