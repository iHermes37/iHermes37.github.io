LLVM_PASS开发

基于LLVM的控制流平坦化的魔改和混淆Pass实战 - 吾爱破解 - 52pojie.cn

开门见山【Pass开发整体流程】

LLVM Pass 是 LLVM 编译框架中用于 分析和转换代码 的核心模块，它允许开发者在编译过程的不同阶段干预中间表示（IR），实现优化、静态分析或代码变形。对 LLVM IR 进行以下操作：

  • 转换（Transformation）：修改 IR（如优化、混淆）。
  • 分析（Analysis）：收集信息（如统计指令数量、检测漏洞）。

设计目标:

  • 模块化：每个 Pass 专注单一任务（如“内联函数”或“删除死代码”）。
  • 可组合性：多个 Pass 可串联形成优化管道。
  • 可扩展性：用户可轻松添加自定义 Pass。

这里首先给出LLVM-PASS开发的整个流程，读者只需知道其大致流程和每一个步骤大底是干什么的即可。

安装 LLVM 与 Clang

既然是基于LLVM的PASS编写自然是需要依赖LLVM提供好的一些库文件。主要有两种方式来安装LLVM

LLVM 项目有多个组件。项目的核心本身称为“LLVM”。它包含处理中间表示并将其转换为目标文件所需的所有工具、库和头文件。工具包括汇编器、反汇编器、位代码分析器和位代码优化器。它还包含基本的回归测试。

类似 C 的语言使用 Clang 前端。此组件将 C、C++、Objective C 和 Objective C++ 代码编译为 LLVM 位代码 – 然后使用 LLVM 将其编译为目标文件。

预编译版本

Ubuntu 20.04 默认仓库中提供的 LLVM/Clang 版本为 LLVM 10。 所以读者如果需要安装更高的版本，有两种方法：

  1. 使用sudo apt install llvm-x.y clang-x.y来指定安装当前源可获取的版本。
  2. 想要安装最新版本，可以根据 apt.llvm.org这个网站列出的方法配置源并安装。

编写 LLVM Pass

编写 LLVM Pass — LLVM 21.0.0git 文档 - LLVM 项目

这里是本文的重点，将在下面重点介绍，在此之前，需要知道简单知道LLVM Pass是什么？以及它的输入输出是什么？

  • LLVM Pass框架是整个LLVM 提供给用户用来干预代码优化过程的框架，也是我们编写代码混淆工具的基础。
  • 编译后的LLVM Pass通过优化器opt进行加载，可以对LLVM IR中间代码进行分析和修改，生成新的中间代码。

输入输出

LLVM Pass 模块的输入和输出是 LLVM 中间表示（IR），但具体形式和处理方式取决于 Pass 的类型（分析、转换、函数/模块级等）。

可以看到pass知识编译系统中的一个模块，只需直接通过Module、Function、BasicBlock、Instruction 等 ，就可以全局直接获取系统知道IR

输入

所有 LLVM Pass 的输入都是 LLVM IR，但 IR 的粒度取决于 Pass 的作用范围：

  • 模块级 Pass（ModulePass）：输入是整个模块（llvm::Module

    ），包含全局变量、函数、元数据等。

  • 函数级 Pass（FunctionPass）：输入是单个函数（llvm::Function），包括其基本块（BasicBlock）和指令（Instruction）。

  • 循环级 Pass（LoopPass）：
    输入是循环结构（llvm::Loop），需先通过循环分析获取。

输入的数据结构：

类名 作用 示例用法
llvm::Module 代表整个编译单元（如一个 .c 文件） Module::getFunctionList()
llvm::Function 单个函数定义 Function::arg_begin()
llvm::BasicBlock 基本块（无分支的指令序列） BasicBlock::getTerminator()
llvm::Instruction 单条 IR 指令（如 add、call） Instruction::getOpcode()

输出

Pass 的输出类型取决于其功能类别：

(1) 分析 Pass（Analysis Pass）

输出：生成 分析结果（如数据流信息、依赖关系等），供其他 Pass 使用。

示例：DominatorTree（支配树分析），AliasAnalysis（别名分析）

(2) 转换 Pass（Transformation Pass）

输出：修改后的 新 IR（可能改变原 IR 结构）。

示例操作：删除无用代码（-adce），内联函数（-inline）

(3) 工具类 Pass

输出：可能生成辅助信息（如优化报告、性能计数器等）。

示例：-time-passes（统计各 Pass 耗时）和-print-after-all（打印 IR 修改过程）

graph LR
    A[输入IR] --> B{Pass类型}
    B -->|分析Pass| C[生成分析结果]
    B -->|转换Pass| D[输出修改后的IR]
    B -->|工具Pass| E[生成报告/统计]
    C --> F[其他Pass使用]
    D --> G[后续优化或代码生成]

编译LLVM Pass

在编写完Pass后，我们需要编译它，得到一个动态库。以便后面使用优化器 opt 将处理中间代码，生成新的中间代码或者直接生成可执行文件

在这里又有三种方式：

方法 优点 缺点 适用场景
与LLVM一起编译 深度集成，调试方便 编译耗时极长（30分钟~数小时） 修改LLVM核心代码或长期开发
CMake单独编译 快速编译（秒级），工程化管理 需要配置CMakeLists.txt 90%的日常Pass开发
命令行单独编译 无需配置构建系统 难以管理依赖，项目越大越混乱 快速原型验证或小型测试

大名鼎鼎的Ollvm便是维护一个定制化LLVM分支修改LLVM核心代码，定制pass执行的。【Pass代码需要存放在llvm/lib/Transforms文件夹中】

Pass开发具体

在前面讲解了pass开发的一般流程，这里将具体讲解pass开发中的一些思想和具体细节。首先先介绍在LLVM旧版中必须要实现的方法然后看看新版中的不同，最后讲述编写PASS的一些个人总结和基本思想

LLVM的核心是一个多阶段流水线编译器，Pass是流水线中的基本处理单元。所有优化和分析都通过Pass实现，因此必须满足LLVM系统的以下要求：

  • 统一管理：需注册到PassManager才能被调度
  • 标准化接口：必须实现特定方法供系统调用
  • 生命周期控制：明确Pass的生效范围和资源管理

所有Pass必须的共同要素

1. 头文件包含

2. 输出依赖声明

3. 编译要求

必须编译为动态库（.so/.dll），链接LLVM核心库（LLVMCore、LLVMSupport等）

机制

LLVM通过基类指针统一管理所有Pass，虚函数runOnXxx是Pass的执行入口（如runOnFunction、runOnModule），为我们知道llvm优化过程中一般有不止一个Pass,为例管理这些Pass。LLVM用ID识别Pass类型，用于Pass依赖分析和去重。而注册宏：static RegisterPass<MyPass> X("opt-name", "Description", false, false);则是为了：

参数 作用 与LLVM系统的关联
“opt-name” 定义opt -opt-name命令行参数 将Pass绑定到命令行接口
“Description” Pass的元信息描述 显示在opt -help中
false 是否只分析不修改IR 影响Pass调度器的优化决策

    C --> D[构建PassPipeline]
    D --> E[PassManager执行run]
    E --> F[Pass::run + AnalysisManager交互]

必须的插件注册

机制

LLVM新架构采用CRTP模式（编译期多态）,run()返回PreservedAnalyses告诉系统哪些分析结果仍有效。而插件注册机制则提供了如下便利：

  • 动态加载：允许Pass独立编译为.so文件
  • 模块化设计：通过回调函数延迟注册，降低耦合度
  • 版本控制：LLVM_PLUGIN_API_VERSION确保ABI兼容
要素 Legacy PM New PM 改进目的
注册方式 静态注册（编译期） 动态插件（运行时） 支持独立编译和热加载
接口设计 虚函数+继承 CRTP+概念约束 减少运行时开销
依赖管理 显式声明getAnalysisUsage() 通过AnalysisManager自动处理 简化Pass间依赖关系
返回值 bool（是否修改IR） PreservedAnalyses位掩码 更精确的优化信息保留
