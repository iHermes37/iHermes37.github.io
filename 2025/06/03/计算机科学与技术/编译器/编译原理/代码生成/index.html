代码生成

有时候直接直接翻译成汇编指令较为困难，认为的拆成几个单独的步骤和模块。

如果简化下中间只翻译一次就得到汇编指令，这种简化后的可以直接叫代码生成器。

代码生成的任务

代码的等价性非常重要，源程序想输出4，编译器处理后的汇编代码不能是输出5。当然，现在像gcc编译器也有一些bug，会将代码翻译错。

分配计算资源

答：变量可以放在内存也可以放在寄存器中，当然放在堆中也是可以的，给变量直接malloc一个空间。但是这样做不太符合变量本身的生命周期，例如函数内部的变量放在堆中就显得不合适的，因为堆中的生命周期是比较长的。一个好的编译器应该尽可能的将变量放置在寄存器中，因为寄存器是高速的。如何将成千上万个变量尽可能的存储在那少的可怜的寄存器中，这是寄存器分配要研究的内容。

栈式计算机代码生成

栈式计算机（Stack Machine）是一种基于操作数栈的计算机模型，其指令通过隐式操作栈顶数据完成计算。Java 虚拟机（JVM）是栈式计算机的典型代表。【栈式计算机的缺点是执行效率低，目前已经没有人用在物理机上了】

栈式计算机的核心特征

特性 说明 对比寄存器计算机
隐式操作数栈 所有计算通过栈顶的压入（push）和弹出（pop）完成，无需显式指定操作数地址。 寄存器计算机需显式指定寄存器（如 add eax, ebx）。
指令简洁 单指令完成操作（如 iadd 直接弹出栈顶两元素相加）。 需多指令加载数据到寄存器再运算（如 mov, add）。
无寄存器依赖 不涉及物理寄存器分配问题，代码可移植性高。 需处理寄存器不足时的溢出（Spill）问题。
执行效率较低 频繁的栈内存访问导致速度慢于寄存器操作。 寄存器直接操作数据，效率更高。

举例说明

这里以JVM举例说明

JVM 的执行依赖以下关键数据结构：

  • 操作数栈（Operand Stack）：存放指令操作的临时数据（类似CPU的通用寄存器，但用栈实现）。
  • 局部变量表（Local Variables）：存储方法的参数和局部变量（通过索引访问，类似寄存器）。
  • 字节码指令集：定义如何操作栈和局部变量表。

字节码指令示例

指令 作用 栈状态变化（执行前 → 执行后）
iconst_1 将整数1压入操作数栈。 [] → [1]
iload_0 将局部变量表第0个槽位的值压栈。 [x] → [x, val0]
iadd 弹出栈顶两整数相加，结果压栈。 [a, b] → [a+b]
istore_2 将栈顶值存入局部变量表第2个槽位。 [x] → []（局部变量表槽位2更新为x）

LLVM IR 的寄存器计算机特征

特性 说明 示例
显式寄存器操作 LLVM IR 使用无限多的虚拟寄存器（如 %1, %2），指令直接操作这些寄存器。 %sum = add i32 %a, %b（%a, %b, %sum 均为寄存器）
无栈操作指令 没有显式的栈操作指令（如 push/pop），内存访问通过 load/store 完成。 %val = load i32, i32* %ptr（非栈操作）
SSA 形式 静态单赋值（SSA）要求每个寄存器只赋值一次，天然契合寄存器架构。 %x = add i32 1, 2 %y = mul i32 %x, 3（%x 不可重复定义）
三地址码风格 大多数指令是“二元操作 + 结果寄存器”的形式，符合寄存器机器的指令集设计。 %res = sub i32 %a, %b（类似 SUB %res, %a, %b）

LLVM 代码生成的实现

  1. 前端生成 IR：将源代码转换为 LLVM IR（寄存器形式），例如 Clang 将 a + b 转为 %1 = add i32 %a, %b。

  2. 后端处理 IR：

    • 寄存器分配：将无限虚拟寄存器映射到目标机器的有限物理寄存器（如 x86 的eax,ebx）。

      • 使用算法如线性扫描或图着色算法。

    • 栈帧处理：当物理寄存器不足时，将溢出（Spill）的变量保存到栈中（此时才涉及栈操作）。

      例如：mov [ebp-4], eax（将 eax 溢出到栈）。

  3. 目标代码生成：生成具体架构（如 x86、ARM）的指令，仍保持寄存器操作风格。

LLVM 指令集详解

LLVM IR 的指令集是 LLVM 中间表示的核心部分，它定义了所有可以在 IR 中执行的操作。这些指令可以分为多个类别，包括算术运算、内存操作、控制流、函数调用等。

1. 指令分类概览

类别 主要指令 用途
算术运算 add, sub, mul, udiv, sdiv, urem, srem, fadd, fsub, fmul 整数和浮点数的基本算术运算。
位运算 shl, lshr, ashr, and, or, xor 移位和逻辑位操作。
比较指令 icmp, fcmp 整数和浮点数的比较（如 eq, ne, sgt, ult 等）。
内存操作 alloca, load, store, getelementptr 栈分配、内存读写和指针计算。
类型转换 trunc, zext, sext, fptrunc, fpext, fptoui, fptosi, uitofp 整数和浮点数之间的类型转换。
控制流 br, switch, ret, unreachable 分支、跳转和函数返回。
函数调用 call, invoke, ret 调用函数和处理异常。
聚合操作 extractvalue, insertvalue 从结构体或数组中提取/插入字段。
向量操作 extractelement, insertelement, shufflevector SIMD 向量操作。
原子操作 cmpxchg, atomicrmw, fence 多线程同步和原子操作。
其他指令 phi, select, va_arg, landingpad SSA 的 phi 节点、条件选择、可变参数处理、异常处理等。

2. 核心指令详解

(1) 算术运算指令

指令 语法 说明
add %res = add i32 %a, %b 整数加法（无符号溢出行为取决于语义）。
fadd %res = fadd float %a, %b 浮点数加法（遵循 IEEE 754）。
sub %res = sub i32 %a, %b 整数减法。
mul %res = mul i32 %a, %b 整数乘法。
udiv %res = udiv i32 %a, %b 无符号整数除法（除零行为未定义）。
sdiv %res = sdiv i32 %a, %b 有符号整数除法（向零舍入）。
urem %res = urem i32 %a, %b 无符号整数取余。
srem %res = srem i32 %a, %b 有符号整数取余（符号同被除数）。

3. 指令集的特点

  1. 强类型：所有操作数必须显式声明类型（如 i32, float*）。
  2. SSA 形式：变量只能赋值一次（通过 phi 节点处理控制流合并）。
  3. 显式内存访问：必须通过 load/store 访问内存。
  4. 平台无关：指令集抽象了硬件细节，后端负责生成目标代码。
