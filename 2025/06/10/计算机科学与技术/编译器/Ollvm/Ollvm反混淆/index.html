模拟执行与符号执行

unicorn模拟执行：核心在于找到全部真实块，之后从首个真实块使用unicorn模拟执行混淆的程序，直到运行到下一个真实块，这里需要注意的是真实块中的分支指令需要单独处理。

angr符号执行解决控制流平坦化：通过符号执行来确定真实块之间的前后关系，真实块之间的前后关系又分为两类：

  1. 有一个确定的后继块
  2. 有两个后继块，跳转到哪个由某个条件决定

第一种情况非常好处理，我们从一个真实块A开始符号执行，直到碰到下一个真实块B，那么我们就确定了一对基本块的执行顺序A→B。

第二种情况我们需要先令该条件为True，然后进行符号执行，得到一个后继真实块的地址；再令条件为False，然后进行符号执行，得到另一个后继真实块的地址。思路比较简单，但代码写起来会稍微有点复杂。

Unicorn（模拟执行） 和 Angr（符号执行） 在解决 OLLVM 控制流平坦化时的核心思路 高度一致，均围绕 识别真实块 和 恢复真实控制流 展开

核心思路的一致性:

步骤 Unicorn 模拟执行 Angr 符号执行
目标 恢复被平坦化隐藏的真实控制流 同左
关键方法 动态执行找到真实块间的跳转关系 符号化分析推导真实块间的可达性
处理分支 强制设置分支条件，观察执行路径 分情况约束条件（True/False）求解路径
输出结果 真实块间的直接跳转边（A→B） 同左

共同逻辑：

  1. 识别真实块：过滤掉 Dispatcher 和虚假块，保留实际有意义的代码块。
  2. 恢复跳转关系：通过执行（动态或符号化）确定真实块之间的前后继关系。
  3. 重建 CFG：基于跳转关系生成简化后的控制流图。

技术实现的差异:

维度 Unicorn Angr
执行方式 动态执行（具体输入驱动） 符号执行（抽象输入覆盖所有路径）
速度 较快（单次执行） 较慢（需求解约束）
路径覆盖 依赖输入，可能遗漏路径 理论上覆盖所有可行路径
分支处理 需手动干预分支条件 自动分情况（True/False）探索
适用场景 已知输入或需快速验证 复杂分支或需全路径分析
工具链 轻量级，适合嵌入自定义工具 重量级，适合复杂分析

1. IDA Python 的核心作用

IDA Python 是 IDA Pro 反汇编器内置的脚本接口，允许用户通过 Python 脚本自动化执行以下任务：

  • 静态分析：遍历函数、基本块、指令，提取控制流图（CFG）、交叉引用（Xrefs）等。
  • 动态修改： Patch 二进制代码、修复符号、修改反汇编结果。
  • 自动化处理：批量重命名变量、识别模式（如加密常量）、修复混淆代码。
  • 插件开发：扩展 IDA 功能（如自定义反混淆算法）。

典型应用场景：

  • 自动化分析恶意软件
  • 逆向工程中的重复性任务（如重命名函数）
  • 解混淆（OLLVM、VMProtect 等）
