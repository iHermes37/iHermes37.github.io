初识JNI

Java 代码在编译后是 **字节码 (.class / .dex)**，不是机器码。它不能直接跑在 CPU 上，必须通过 JVM/ART(虚拟机运行时) 去解释或 JIT/AOT 编译成机器码。所以 Java 世界必须有 运行时（ART/Dalvik/JVM）。

  • Java → C++ 调用：Java 通过 JNI 找到 native 方法，交给 CPU 直接跑 C++ 实现。
  • C++ → Java 调用：C++ 并不是直接执行 Java，而是通过 JNI API 把请求交给虚拟机，由虚拟机帮忙执行。

C++ 如何“调用” Java？这里的“调用”，其实并不是绕过虚拟机直接执行 Java，而是：C++ 通过 JNI 接口，向虚拟机发出请求，让虚拟机去执行对应的 Java 方法。

换句话说：C++ 并不是直接运行 Java 字节码，而是通过 JNIEnv* 这样的 虚拟机代理对象，告诉虚拟机：“去执行某个 Java 类里的方法”，虚拟机再去解释/编译运行 Java 代码，并返回结果给 C++

  • JNI 是桥梁：它让 Java 和 C++/C 互相调用。
  • **Java → C++**：JNI 找到 native 方法的实现，执行 C++ 代码。
  • C++ → Java：JNI 通过 JNIEnv 调用 Java 方法，由 ART 执行。

ArtMethod

  • 普通 Java 方法：有 ArtMethod，指向 Dex/机器码。
  • Java native 方法：有 ArtMethod，指向 JNI stub，再跳到 C++。
  • 纯 C++ 函数（不和 Java 绑定）：没有 ArtMethod，因为它不在 Dex/ART 的管理范围内。

换句话说： ArtMethod 的存在前提是 方法必须在 Java 层声明。
C++ 世界里的函数本身不会生成 ArtMethod，它只是在被 Java 调用时，通过 JNI 桥接，成为某个 native Java 方法的“后端实现”。

JavaVM以及JNIEnv对象

JavaVM 对象

  • 表示一个 Java 虚拟机实例。
  • 进程里可以有多个虚拟机实例（在 Android 中通常只有一个 ART 虚拟机）。
  • 你可以把它理解为 “JVM 管理员/总控入口”。

主要职责：

  • 生命周期管理：创建 / 销毁 JVM。
  • 线程附加/分离：C++ 线程如果要调用 Java 方法，必须先 AttachCurrentThread 绑定到 JVM，获取对应的 JNIEnv。
  • 跨线程入口：因为 JNIEnv 是线程局部的（thread-local），如果换线程了，就得通过 JavaVM 重新拿 JNIEnv。

JNIEnv 对象

  • 代表 当前线程和 JVM 的会话接口。
  • 每个线程都有自己独立的 JNIEnv*，不能跨线程使用。
  • 它就像 “虚拟机的遥控器”，提供大量函数指针，让你在 C++ 里能操作 Java 世界。

主要职责：

  • 类和方法操作：FindClass、GetMethodID、CallObjectMethod 等。
  • 对象管理：创建对象 (NewObject)、管理引用 (NewGlobalRef、DeleteLocalRef)。
  • 异常处理：检查和清理 Java 异常 (ExceptionCheck、ExceptionClear)。

Java → C++

  • Java 调用一个 native 方法时，JVM 会把 JNIEnv* 和 jclass/jobject 传给 C++。
  • JNIEnv 作用：让 C++ 可以继续调用回 Java，操作虚拟机资源。
  • JavaVM 一般不用管，因为 JVM 已经帮你准备好了 JNIEnv*。

C++ → Java

  • C++ 线程想主动调 Java，需要先通过 JavaVM 附加线程，拿到 JNIEnv*。
  • JavaVM 作用：线程注册到 JVM，获取 JNIEnv。
  • JNIEnv 作用：具体执行 Java 方法、操作对象。

JavaVM = JVM 总控入口，线程注册处（类似“系统登录入口”）。

JNIEnv = 当前线程的 JVM 代理接口（类似“系统 API 遥控器”）。

**Java → C++**：JVM 自动给你 JNIEnv，直接干活。

C++ → Java：必须先通过 JavaVM 登录线程，再用 JNIEnv 调用 Java。

Java层如何调用Native层的一个方法

Java层中的native方法如何与Native层中对应的方法相关联

dex2oat编译这个Java native方法后将会生成一段机器码。ArtMethod对象的机器码入口地址会指向这段生成的机器码。这段机器码本身会跳转到这个ArtMethod对象的JNI机器码入口地址。如果这个JNI方法没有注册过（即这个native方法还未和Native层对应的函数相关联），这个JNI机器码入口地址是art_jni_dlsym_lookup_stub。否则，JNI机器码入口地址指向Native层对应的函数。

如果dex2oat没有编译过这个Java native方法，则ArtMethod对象的机器码入口地址为跳转代码art_quick_generic_jni_trampoline。同样，如果这个JNI方法没有注册过，则JNI机器码入口地址为跳转代码art_jni_dlsym_lookup_stub。否则，JNI机器码入口地址指向Native层对应的函数。
