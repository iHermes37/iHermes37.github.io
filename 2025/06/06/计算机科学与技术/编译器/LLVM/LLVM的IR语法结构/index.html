LLVM的IR语法结构

  1. 模块（Module）：LLVM IR的最顶层结构是模块，表示整个程序或库。一个模块由一组全局变量和函数组成。
  2. 全局变量（Global Variables）：LLVM IR允许声明全局变量，它们是在整个程序中可见的。全局变量由其类型、名称和可选的初始值组成。全局变量的声明以@符号开头，后面跟着变量名称和类型信息。例如，@myGlobal = global i32 0表示一个名为myGlobal的32位整数全局变量，初始值为0。
  3. 函数（Functions）：LLVM IR中的函数由函数类型、名称、参数列表和函数体组成。函数类型包括返回类型和参数类型。函数的声明以define关键字开头，后面是返回类型、函数名称、参数列表和函数体。函数体由基本块（Basic Block）组成，每个基本块由一组指令组成。函数体的最后一条指令通常是返回指令。
  4. 基本块（Basic Blocks）：基本块是LLVM IR中的基本执行单元。它由一系列按顺序执行的指令组成，没有分支或跳转指令。基本块以标签（Label）开头，后面是一组指令。基本块可以包含控制流指令，如条件分支和无条件跳转，以便实现程序的控制流程。
  5. 指令（Instructions）：LLVM IR包括各种指令来表示程序的操作。指令由操作码和操作数组成，用于执行特定的操作。LLVM IR支持常见的指令类型，如赋值指令、算术运算指令、逻辑运算指令、条件分支指令、内存访问指令等。指令可以使用变量、常量和表达式作为操作数。

语法结构

模块

语法元素 格式示例 说明
目标三元组 target triple = "x86_64-pc-linux-gnu" 声明目标平台（架构-厂商-系统-ABI）。
数据布局 target datalayout = "e-m:e-p270:32:32-p271:32:32-p272:64:64-f64:32:32..." 定义内存对齐规则和端序。
全局变量声明 @global_var = global i32 42 声明全局变量并初始化。
函数声明 declare i32 @printf(i8*, ...) 声明外部函数（无函数体）。
函数定义 define i32 @add(i32 %a, i32 %b) { ... } 定义函数及其实现。

全局变量

语法元素 格式示例 说明
整数全局变量 @x = global i32 10 32位整数全局变量，初始值为10。
浮点全局变量 @y = global float 3.14 单精度浮点数全局变量。
字符串全局变量 @str = constant [12 x i8] c"Hello\00" 常量字符串（以\00结尾）。
外部全局变量 @ext_var = external global i32 声明外部链接的全局变量（定义在其他模块）。
线程局部变量 @tls_var = thread_local global i32 0 线程局部存储（TLS）变量。

函数

语法元素 格式示例 说明
函数定义 define i32 @add(i32 %a, i32 %b) { ret i32 %sum } 定义函数add，接受两个i32参数，返回i32。
可变参数函数 declare i32 @printf(i8*, ...) 声明可变参数函数（如printf）。
无返回值函数 define void @foo() { ret void } 无返回值函数。
函数属性 define void @foo() nounwind { ... } 添加函数属性（如nounwind表示无异常抛出）。
内联汇编 call void asm "nop", ""() 嵌入汇编代码。

基本块

语法元素 格式示例 说明
带标签的基本块 bb1: 基本块以标签开头，后跟指令序列。
终止指令 br label %bb2 基本块必须以终止指令（如br、ret）结尾。
PHI节点 %val = phi i32 [0, %entry], [%inc, %loop] 根据前驱块选择值（用于控制流合并点）。
调试信息 !dbg !123 附加调试元数据（如行号）。

指令

算术运算

指令 格式示例 说明
加法 %sum = add i32 %a, %b 整数加法（add、fadd）。
减法 %diff = sub i32 %a, %b 整数减法（sub、fsub）。
乘法 %prod = mul i32 %a, %b 整数乘法（mul、fmul）。
除法 %quot = sdiv i32 %a, %b 有符号除法（sdiv/udiv）。

内存操作

指令 格式示例 说明
加载 %val = load i32, i32* %ptr 从内存加载值。
存储 store i32 42, i32* %ptr 将值存储到内存。
分配栈空间 %ptr = alloca i32 在栈上分配内存（返回指针）。

控制流

指令 格式示例 说明
无条件跳转 br label %bb2 跳转到指定基本块。
条件分支 br i1 %cond, label %true, label %false 根据条件跳转。
返回 ret i32 0 从函数返回。
Switch switch i32 %val, label %default [i32 1, label %case1] 多路分支。

逻辑运算

指令 格式示例 说明
与运算 %and = and i32 %a, %b 按位与。
或运算 %or = or i32 %a, %b 按位或。
异或 %xor = xor i32 %a, %b 按位异或。
比较 %cmp = icmp eq i32 %a, %b 整数比较（eq/ne/slt/ult等）。

其他指令

指令 格式示例 说明
函数调用 %ret = call i32 @foo(i32 %arg) 调用函数并获取返回值。
类型转换 %conv = trunc i32 %a to i16 类型转换（trunc/zext/sext/fptoui等）。
向量操作 %vec = add <4 x i32> %a, %b SIMD向量运算。

元数据

语法元素 格式示例 说明
调试信息 !dbg !123 关联调试信息（如!DILocation）。
别名分析 !alias.scope !{!0} 标记内存别名范围。
循环信息 !llvm.loop !{!1} 标记循环属性（如展开次数）。

LLVM IR 类型系统

类型分类 语法格式 示例 说明
整数类型 iN i32, i1 N位整数（i1是布尔型）。
浮点类型 float, double, fp128 float, double 单/双精度浮点数。
指针类型 type* i32*, i8** 指向type的指针（可嵌套）。
数组类型 [N x type] [10 x i32] 固定长度数组（N个type元素）。
向量类型 <N x type> <4 x float> SIMD向量（如SSE/AVX指令）。
结构体类型 {type1, type2, ...} {i32, float} 匿名结构体（字段类型按顺序排列）。
命名结构体 %struct.Name = type {type1, ...} %struct.Point = type {i32, i32} 带名称的结构体（需全局声明）。
函数类型 ret_type (param_types) i32 (i32, float)* 函数指针类型（返回类型 + 参数列表）。
不透明类型 opaque %opaque.type 未定义内容的结构体（用于隐藏实现）。

架构设计

寄存器设计

特性 语法格式 示例 说明
SSA形式 %reg = op type %arg1, %arg2 %sum = add i32 %a, %b 静态单赋值（每个寄存器只赋值一次）。
虚拟寄存器 %vreg %1, %2 编译器临时分配的寄存器（无物理寄存器限制）。
物理寄存器 %eax, %rsp call void asm "mov $0, %eax" 内联汇编中绑定到硬件寄存器。
Phi节点 %val = phi type [val1, %bb1], ... %x = phi i32 [0, %entry] 根据前驱块选择值（控制流合并点）。
全局寄存器 @global_reg @counter = global i32 0 全局变量（存储在内存，非寄存器）。

整体IR架构

层级 组件 关键语法 说明
顶层模块 target triple, target datalayout target triple = "x86_64-..." 定义目标平台和内存布局。
全局变量 @global = type value @MAX = constant i32 100 全局常量/变量（.data/.rodata节）。
函数定义 define ret_type @func(args) {...} define i32 @add(i32 %a, i32 %b) 函数体由基本块组成。
基本块 label: inst1; inst2; ... bb1: %x = add i32 %a, %b 线性指令序列，以终止指令结尾。
指令集 %res = op type %op1, %op2 %cmp = icmp eq i32 %x, 0 包括算术、内存、控制流等操作。
元数据 !tag !value !dbg !123 附加调试信息或优化提示。

关键设计原则

设计原则 LLVM IR实现 优势
强类型系统 显式标注所有操作数的类型 防止类型错误，支持优化。
SSA形式 虚拟寄存器无限，Phi节点处理控制流合并 简化数据流分析，提升优化效果。
分层结构 模块→函数→基本块→指令 模块化设计，便于编译器逐步处理。
平台无关性 target triple定义目标架构 同一份IR可跨平台编译。
