Keystone基本使用

以下是 Keystone 编译工具（Python 版）的使用大全，以表格形式展示其核心功能、API 及示例代码：

1. Keystone 基础配置

功能 API/方法 示例代码 说明
初始化引擎 Ks(arch, mode) ks = Ks(KS_ARCH_X86, KS_MODE_64) 创建 X86-64 汇编引擎实例。
设置语法 ks.syntax = syntax_type ks.syntax = KS_OPT_SYNTAX_INTEL 支持 KS_OPT_SYNTAX_INTEL（默认）或 KS_OPT_SYNTAX_ATT（AT&T 格式）。
设置选项 ks.options = option_flags ks.options = KS_OPT_SYNTAX_NASM 控制汇编行为（如 NASM 兼容模式）。

2. 汇编指令（编码）

功能 API/方法 示例代码 说明
单条指令汇编 ks.asm(assembly, addr=base_addr) encoding, count = ks.asm("mov eax, 1", addr=0x400000) 返回机器码和指令数。addr 用于相对跳转计算。
多条指令汇编 ks.asm(assembly, addr=base_addr) encoding, count = ks.asm("mov eax, 1; ret", addr=0x400000) 用分号或 \n 分隔多条指令。
处理汇编错误 KsError 异常 try: ks.asm("invalid")
except KsError: print("汇编失败")		 捕获无效指令异常。

3. 输出与调试

功能 API/方法 示例代码 说明
获取机器码 ks.asm() 返回的 encoding bytes(encoding) 转换为字节序列，可直接写入内存。
打印十六进制 手动转换 print(" ".join(f"{b:02x}" for b in encoding)) 输出如 b8 01 00 00 00 c3。
指令地址映射 ks.asm() 的 addr 参数 encoding, _ = ks.asm("jmp 0x400000", addr=0x401000) 计算相对跳转偏移量。

4. 多架构支持

架构 初始化参数 示例代码 说明
x86 / x86-64 KS_ARCH_X86, KS_MODE_32/KS_MODE_64 ks = Ks(KS_ARCH_X86, KS_MODE_32) 支持 16/32/64 位模式。
ARM / ARM64 KS_ARCH_ARM, KS_MODE_ARM/KS_MODE_THUMB ks = Ks(KS_ARCH_ARM, KS_MODE_THUMB) Thumb 模式需指定 KS_MODE_THUMB。
MIPS KS_ARCH_MIPS, KS_MODE_MIPS32/KS_MODE_MIPS64 ks = Ks(KS_ARCH_MIPS, KS_MODE_MIPS32) 支持 MIPS32/64。
PowerPC KS_ARCH_PPC, KS_MODE_PPC32/KS_MODE_PPC64 ks = Ks(KS_ARCH_PPC, KS_MODE_PPC64) 支持 32/64 位 PowerPC。

5. 高级功能

功能 API/方法 示例代码 说明
自定义内存引用 结合 addr 参数计算 ks.asm("mov eax, [0x1000]", addr=0x400000) 生成绝对地址引用。
指令别名 手动替换 ks.asm("nop") → ks.asm("mov eax, eax")（某些架构） 部分指令需手动转换。
批量汇编 循环调用 ks.asm() for asm in ["push ebp", "mov ebp, esp"]:
encoding += ks.asm(asm)[0]		 合并多条指令的机器码。

7. 注意事项

  1. 指令兼容性：不同架构的指令集支持可能受限（如 ARM 的某些扩展指令需验证）。
  2. 内存管理：生成的机器码需自行写入可执行内存（如 mmap）。
  3. 性能：频繁初始化 Ks 对象会降低效率，建议复用实例。

通过 Keystone 的 Python API，可灵活实现动态代码生成、漏洞利用开发等高级任务。
