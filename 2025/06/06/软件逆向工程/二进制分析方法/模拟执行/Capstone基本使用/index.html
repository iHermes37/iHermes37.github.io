Capstone基本使用

在 Capstone 引擎中，md.disasm() 返回的 指令对象（i） 是一个包含指令所有信息的结构化对象。以下是其核心组成和关键属性的详细说明：

1. 指令对象（CsInsn）的核心属性

属性 类型 说明
address int 指令的虚拟地址（如 0x400500）。
mnemonic str 指令助记符（如 "mov"、"bl"）。
op_str str 操作数字符串（如 "x0, x1"）。
size int 指令长度（字节数，如 4）。
bytes bytes 指令的原始机器码（如 b"\x01\x00\x80\xd2"）。
operands list[CsOp] 操作数列表（需 md.detail=True 启用）。
groups list[int] 指令所属的功能组（如跳转、调用等，需 md.detail=True）。

2. 操作数对象（CsOp）的组成

当启用 md.detail=True 时，i.operands 返回操作数列表，每个操作数对象包含：

属性 类型 说明
type int 操作数类型（如 ARM64_OP_REG、ARM64_OP_IMM）。
value Union[int, dict] 操作数值：
- 立即数：value.imm（如 0x400500）
- 寄存器：value.reg（如 ARM64_REG_X0）
access int 操作数访问方式（读、写或读写，如 CS_AC_READ）。

常见操作数类型（以 ARM64 为例）：

  • ARM64_OP_REG: 寄存器（如 x0）。
  • ARM64_OP_IMM: 立即数（如 0x100）。
  • ARM64_OP_MEM: 内存引用（如 [x0, #0x10]）。

3. 指令功能组（groups）

i.groups 是一个整数列表，表示指令所属的功能类别。常用组别常量如下：

组别常量 值 说明
CS_GRP_JUMP 1 跳转指令（如 b、bl）。
CS_GRP_CALL 2 调用指令（如 bl）。
CS_GRP_RET 3 返回指令（如 ret）。
CS_GRP_INT 4 中断指令（如 svc）。

5. 实际应用场景

  1. 控制流分析：
    • 通过 groups 识别分支指令（如 CS_GRP_JUMP）。
    • 提取跳转目标（operands[0].value.imm）。
  2. 寄存器追踪：
    • 分析 mov x0, x1 中的源/目标寄存器。
  3. 二进制修改：
    • 通过 bytes 获取原始机器码，用于补丁生成。
  4. 漏洞检测：
    • 检查危险指令（如 str x0, [sp] 可能导致栈溢出）。

总结

Capstone 的指令对象是一个 高度结构化的反汇编结果，包含地址、助记符、操作数、功能组等关键信息。通过解析这些属性，可以实现：

  • 精确的指令级分析（如寄存器/内存访问）。
  • 控制流重建（识别分支、调用、返回）。
  • 二进制自动化处理（如插桩、优化）。

启用 md.detail=True 是获取操作数和功能组信息的前提！
