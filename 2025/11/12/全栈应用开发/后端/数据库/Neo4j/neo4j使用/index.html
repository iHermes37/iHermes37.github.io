neo4j使用

第 1 章 Neo4j的亮点 — neo4j 0.1.0a 文档

结构

与 MySQL 等关系型数据库的“表、行、列”不同，Neo4j 的数据模型基于节点和关系。

概念 类比关系型数据库 描述 示例
节点 表中的一行记录 代表一个实体（Entity） 一个人、一部电影、一个产品、一个城市
关系 外键关联 代表节点之间的连接（Connection），必须有方向和类型 人 -[:看过]-> 电影
属性 列/字段 用于描述节点或关系的键值对 人的 姓名、电影的 标题、关系的 评分
标签 表名 对节点进行分类，一个节点可以有多个标签 给节点打上 Person 或 Movie

关系

在 Neo4j 中，所有的节点和关系都是完全自定义的。所有关系类型都需要根据自己的业务需求来设计和创建。这里给出实践示例：

特性维度 分类 / 类型 描述 示例
方向性 有向关系 关系具有明确的起点和终点，用 -> 或 <- 表示。 (A)-[:FOLLOWS]->(B) (A 关注 B)
无向关系 查询时不指定方向，匹配任意方向的关系。 (A)-[:FRIENDS_WITH]-(B) (A 和 B 是朋友)
类型与语义 结构关系 描述节点之间的结构性连接，如组织架构。 (员工)-[:BELONGS_TO]->(部门)
交互关系 描述节点之间的行为或事件，如交易、点击。 (用户)-[:PURCHASED]->(商品)
语义关系 描述现实世界中的抽象联系，如归属、相似。 (药品)-[:TREATS]->(疾病)
数量与复杂度 一对一 一个节点只与另一个节点有此种关系。 (人)-[:HAS_SSN]->(社保号)
一对多 一个节点与多个节点有此种关系。 (母亲)-[:HAS_CHILD]->(孩子)
多对多 多个节点与多个节点有此种关系。 (学生)-[:ENROLLED_IN]->(课程)
动态性 静态关系 创建后一般不变化的长期稳定关系。 (城市)-[:LOCATED_IN]->(国家)
动态关系 带有时间戳等属性，表示会变化的状态。 (用户)-[:WORKED_FOR {from:2020, to:2023}]->(公司)
层级与递归 层级关系 描述树状或层级结构，如包含、汇报。 (员工)-[:REPORTS_TO]->(经理)
递归关系 同一类型的节点通过同种关系形成链或环。 (人)-[:CHILD_OF]->(人)

关系深度与模式分类

这个表格专门描述在查询中，如何通过关系模式匹配不同深度的连接。

关系模式 Cypher 语法 描述 示例场景
单跳关系 -[:REL_TYPE]-> 直接连接，一度关系。 查找用户的直接朋友。
多跳关系 -[:REL_TYPE*2]-> 固定跳数的关系路径。 查找朋友的朋友（二度人脉）。
可变长度关系 -[:REL_TYPE*3..5]-> 跳数在一个范围内。 查找 3 到 5 度的人脉。
任意长度关系 -[:REL_TYPE*]-> 任意跳数的关系路径（直到图的尽头）。 查找所有的祖先或后代。
最短路径 shortestPath(...) 查找两个节点间的最短关系路径。 计算两个人之间的最短介绍路径。

关系属性详解

关系本身也可以拥有属性，这使得关系能够承载更丰富的信息。

属性类型 描述 示例
权重/强度 表示关系的强度或容量。 -[:RATED {score: 5}]->
时间性 表示关系的开始、结束或发生时间。 -[:WORKED_FOR {from: 2020, to: 2023}]->
成本/距离 表示关系的成本、距离或难度。 -[:ROAD_TO {distance_km: 150}]->
状态与元数据 描述关系本身的状态或创建信息。 -[:FRIENDS_WITH {status: 'active', created: timestamp()}]->

Cypher 查询语言

Cypher 是 Neo4j 图数据库的声明式查询语言，它的设计非常直观，核心思想是 “让查询语句看起来像它所描述的图模式”。Cypher 的核心在于描述想要查找的图模式。而不需要告诉数据库“如何”找到数据，只需要描述数据“长什么样”。
