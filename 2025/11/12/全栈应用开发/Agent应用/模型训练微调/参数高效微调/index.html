参数高效微调

背景

微调的根本目的，是将通用模型转变为领域专家。可以将一个在大规模通用数据（如互联网文本、海量图像）上预训练好的模型（例如BERT、GPT、ResNet）看作一位“博学的通才”。它掌握了世界的基础规律——语言的结构、图像的构成、常见的逻辑。然而，它并不天然适用于你的特定任务。微调就是这个“因材施教”的关键过程，其核心价值体现在三个层面：

  1. 实现知识迁移，突破数据壁垒：对于绝大多数企业和开发者而言，从头训练一个大型模型（需要海量数据、巨额算力和深厚技术）是遥不可及的。微调允许我们站在巨人的肩膀上，利用预训练模型已经学到的强大“基础知识”，仅使用自己有限的、高质量的领域数据，对其进行“点拨”和“专业化训练”。这相当于用建造一栋平房的成本和时间，改造出了一栋摩天大楼，极大地降低了AI应用的门槛。
  2. 获得任务定制化的最优性能：尽管存在“提示工程”等不更新模型权重的方法，但对于复杂任务（如法律条款分析、医疗报告生成、特定风格的图像创作），零样本或小样本学习往往无法达到生产级要求的精度和可靠性。微调通过调整模型内部的数百万乃至数十亿个参数，让模型的“思维模式”从根本上适应新任务的分布和需求，从而在特定任务上达到当前技术路径下的性能上限。这是将模型潜力转化为实际商业价值的核心手段。
  3. 注入领域知识，规范模型行为：通用模型可能会产生不受控制、甚至有风险的输出。通过微调，我们可以将特定的领域知识、行业术语、行为准则和安全边界“内化”到模型之中。例如，让客服机器人熟练掌握产品手册，让代码助手遵循公司的编程规范，让诊断模型只输出符合医学指南的建议。这不仅是提升性能，更是对模型行为进行精准控制和对齐的必要过程。

需要注意的是:

微调不是无米之炊。至少需要数百个（对于大模型可能需要数千至上万个）高质量的、标注好的数据样本，才能有效引导模型而不过度拟合。这里的“高质量”指数据能准确代表你的业务场景且标注一致。如果拥有这样的数据集，微调就能将其价值最大化。

此外当需要对模型的输出风格、格式或价值观进行严格控制时：如果应用场景对输出的格式、口吻、安全性有严格要求（例如，要求模型始终以JSON格式输出、采用正式严谨的公文风格、或坚决避免某些不安全的话题），微调是实现这种“行为对齐”的最有效方式。通过使用符合要求的样本进行训练，可以将特定的输出规范“刻入”模型的行为模式中，这是简单提示难以做到的。

长文本 & 知识库 & 微调

长文本处理、知识库构建与模型微调是构建大模型应用时不同层级、不同目的的技术手段，它们分别解决了“模型能看多长”、“模型能知道什么”和“模型如何思考与表达”的问题。

  • 长文本能力是模型的基础硬件属性，决定了模型单次交互的“信息接收窗口”大小，如同一个人的短期记忆容量；
  • 知识库是模型的外部长期记忆体，通过检索增强生成技术为模型提供实时、准确、私有的知识来源，弥补了模型内部知识可能过时、不完整或存在幻觉的缺陷；
  • 而微调则是针对模型本身“大脑”的深度改造，通过训练改变其权重参数，从而优化其任务执行能力、输出风格或对特定领域知识的理解深度。

简而言之，长文本决定了模型能“阅读”多长的资料，知识库决定了模型能“参考”哪些资料，而微调则决定了模型如何“消化”这些资料并形成特定的“思维模式”来解决问题。三者并非互斥，在高级应用中往往协同工作：一个强大的AI助手可能基于具备长文本能力的基座模型，通过微调使其精通法律领域的分析与表达，并连接一个实时更新的法律条文知识库来提供精准服务。

SFT

SFT 是全称 Supervised Fine-Tuning 的缩写。如上图所示，它通常是优化一个通用基座模型的第一步。其过程类似于“师傅带徒弟”，我们为模型提供大量高质量的 指令-输出对，通过传统的监督学习方式，让模型学习如何根据指令生成合适的回答。

一个只在海量文本上预训练出来的基座模型，它虽然拥有丰富的知识，但并不知道如何与人类进行对话、如何遵循指令、或者如何扮演一个特定的角色。SFT 就是激活模型的能力，教会它“如何说话”和“如何回答问题”。

RL

在AI领域，这里的RL通常特指 RLHF。其核心思想是：我们不再提供标准答案，而是让模型生成多个答案，然后由一个奖励模型 对这些答案进行评分，判断哪个更好、更符合人类偏好。然后，利用这个评分作为强化学习的奖励信号，去调整模型的策略，让它未来更倾向于生成高分的答案。

SFT 教会的只是“如何回答问题”，但无法保证答案一定是“有用的、安全的、无害的”。人类的偏好非常复杂微妙，很难用单一的“标准答案”来覆盖。RLHF 就是为了解决这个问题：对齐。它将模型的输出与复杂、多维度的人类价值观进行对齐，使模型变得不仅能力强，而且“情商高”、安全可靠。

PEFT

PEFT 是 Parameter-Efficient Fine-Tuning 的缩写，中文译为 参数高效微调。它是一种旨在仅微调预训练模型中极小一部分参数，同时达到与全参数微调相媲美的性能的技术范式。

  • 全参数微调：就像为了学习一门新专业（如医学），决定重学所有知识（从小学语文数学到大学物理化学），这需要巨大的时间和精力（计算资源）。
  • PEFT：一名工程师，现在想转行做医学工程师。不需要重学所有医学知识，而是只学习将工程知识与医学领域连接起来的关键桥梁知识（如医学术语、法规、基本原理）。这效率极高，且原有的工程知识（预训练模型的核心能力）得以保留。

全参数微调相对于把预训练模型所有参数都当成可学习参数，直接在下游任务上继续训练。而在前言我们也提到过，LoRA的核心想法：不需要所有的参数同时训练，冻结原始参数，只训练一个小的 ΔW。

Q1：为什么可以直接通过一个低秩矩阵训练就可以达到好的效果？

  • 预训练模型已经学到大部分通用能力，下游只需要“小幅度方向修正”，本身就低维；
  • 实证上，全参微调的权重差分 ΔW 做 SVD 会发现谱衰减很快，“有效秩”不高；
  • LoRA 把更新限制在低秩子空间里，相当于一种强正则，在有限数据下往往更不容易过拟合。

更直观地类比，可以把全参数更新看成一个高维矩阵，而真正“有用”的信息只集中在少数几个方向上，就像 PCA 里只有前几维主成分承载了大部分方差；其余大量维度对应的只是很小的奇异值或噪声成分，对任务几乎没有贡献，甚至会带来过拟合。

LoRA

[2106.09685] LoRA: Low-Rank Adaptation of Large Language Models

LoRA（Low-Rank Adaptation，低秩自适应）是一种高效的大模型微调技术，专门用于解决大语言模型（如GPT、LLaMA等）全参数微调时计算成本高、存储开销大的问题。

核心思想：冻结预训练模型权重，只训练少量新增的低秩矩阵参数。

  1. 传统微调的问题：大模型参数巨大（数十亿到万亿），训练所有参数需要大量GPU内存和算力。每个任务都需要保存完整的模型副本，存储成本高。

  2. LoRA的解决方案：

    • 冻结原始权重：保持预训练模型的参数不变。
    • 引入低秩分解：对于模型中的某个权重矩阵 $ W \in \mathbb{R}^{d \times k} $，新增两个小矩阵 $ A \in \mathbb{R}^{d \times r} $ 和 $ B \in \mathbb{R}^{r \times k}$（$ r \ll \min(d,k) $）。
    • 微调过程：仅训练 ( A ) 和 ( B )，更新时权重变化为 $Delta W = BA $，总权重为 $ W + \Delta W $。

    公式表示：
    $$
    h = Wx + \Delta Wx = Wx + BAx
    $$

    其中 ( r )（秩）通常很小（如8、16），大大减少可训练参数。

主要优势:

优势 说明
高效计算 可训练参数减少10,000倍以上，训练速度快，GPU内存需求低。
轻量存储 只需保存新增的LoRA权重（几MB ~ 数百MB），而非整个模型（数十GB）。
易于部署 多个任务可共享同一个基础模型，只需切换不同的LoRA适配器。
避免灾难性遗忘 基础模型参数固定，保留原始知识。

典型应用步骤

  1. 选择基础模型（如LLaMA、ChatGLM）。
  2. 注入LoRA模块：在Transformer的注意力层（Q、K、V、O）或全连接层旁添加低秩矩阵。
  3. 训练：只训练新增的LoRA参数，使用任务数据（如指令数据、领域文本）。
  4. 推理：将训练好的LoRA权重与基础模型合并，或动态加载适配器。

给哪些层加 LoRA

如何加，加在哪里？LLM 里最常见的做法是：只给 Attention 加 LoRA，尤其是：Self-Attention 里的 Q / K / V / O projection；或者只在 Q / V 上加 LoRA（常见“轻量实践方案”）。

原因很直观：

  • 注意力层直接决定“信息流怎么路由”，对模型行为影响最大；
  • FFN（MLP）层更多是“非线性变换”，某些任务上边际收益会略逊于 Attention；
  • 在 GPU 资源有限的情况下，优先把 LoRA 参数预算砸在 Q/V 上，性价比高。

Q6：在 LLaMA / Qwen 这类 LLM 上做 LoRA，你会加在哪些权重矩阵上？

最常见做法是在 Self-Attention 的 Q、K、V、O 投影矩阵 上加 LoRA； 许多实践发现，只在 Q / V 上加 LoRA 就能取得不错效果，是一种很好的轻量配置； 给 FFN 的 Linear 加 LoRA 也可以，但在算力有限的情况下，更倾向先把预算给 Attention，再视任务表现决定是否扩展到 FFN。

QLoRA

LoRA主要解决了参数效率问题，而没有考虑到模型的大小以及推理阶段的速度和显存占用。然而现代的大模型体积过于庞大，就拿我现在用的Nvidia A100 DGX服务器、24G内存显卡来说，如果使用LoRA进行微调Llama3-8B模型，肯定仍然会出现内存溢出问题。

为了进一步压缩和加速微调后的LoRA模型，我们可以在此基础上应用量化（Quantization）技术。

那么，什么是量化？其实很简单，量化是用较低的数值精度（如int8、int4）去表示原本的浮点数（float32），从而减小模型体积。量化可以在模型训练完成后进行（Post-training Quantization，PTQ），也可以在训练过程中同步进行（Quantization-aware training，QAT）。

如何做到的？

原始模型中的权重通常以float32格式存储，每个权重需要32位来表示。在量化过程中，这些浮点数被转换成整数，且使用的位数更少。例如：

  • int8量化：每个权重用8位整数表示。这意味着每个权重的取值范围限制在-128到127之间（对于有符号整数）。
  • int4量化：每个权重用4位整数表示，取值范围是-8到7之间（对于有符号整数）。

这种转换通常涉及到缩放和偏移操作，以保持量化前后的值尽可能接近。例如，浮点数乘以一个缩放因子（scale factor）后四舍五入到最近的整数，然后可能还需要加上一个偏移量（zero point）以匹配整数表示的范围。

通过使用更少的位来表示每个权重，量化可以显著减少模型的存储需求。以一个简单的例子说明。

  • 假设一个模型有1百万个权重（当然真实的大模型的参数数量远远超过1百万）。
  • 在float32格式中，这些权重总共需要32位 x 1,000,000 = 32,000,000位，即4MB的存储空间。
  • 如果使用int8量化，相同的权重只需要8位 x 1,000,000 = 8,000,000位，即1MB的存储空间。
  • 如果进一步使用int4量化，存储需求降至4位 x 1,000,000 = 4,000,000位，即0.5MB。

目前，最为流行的量化技术就是LoRA和量化相结合的变体，叫做QLoRA。

QLoRA（Quantized Low-Rank Adaptation） 是LoRA的量化增强版，由华盛顿大学团队于2023年提出。它在LoRA基础上引入了量化技术，实现了极致的内存效率，使得在单张消费级GPU（如24GB RTX 4090）上微调超大规模模型（如65B参数）成为可能。

核心创新：4位量化 + LoRA

QLoRA的核心可以概括为以下三层技术叠加：

技术层 作用 具体实现
4位NormalFloat量化 压缩模型权重，大幅减少内存占用 将预训练模型量化为4位精度（而非16位）
双量化 进一步压缩量化常数 对量化参数再次量化，节省额外内存
Paged Optimizers 防止显存溢出 像电脑内存分页一样，将优化器状态临时转移到CPU内存

关键公式：
训练时前向传播公式与LoRA类似，但基础权重 ( W ) 被量化为4位：
$$
h = \text{dequantize}(W_{4bit})x + BAx
$$
其中 $ W_{4bit} $ 是4位量化的权重，只在计算时反量化为16位（计算时精度不损失）。

工作流程（三步量化）

  1. 预训练模型 → 4位量化

    使用改进的4位数据类型（NF4，专为正态分布权重设计），模型大小减少约4倍（如65B模型从130GB → ~32GB）

  2. 训练时动态反量化

    仅在使用时将4位权重临时反量化为16位进行计算，计算后立即释放，不占用持久显存，保持数值精度，性能接近全精度微调

  3. 仅优化LoRA参数

    与LoRA相同，只训练少量低秩矩阵（A、B），优化器状态也因参数少而内存占用低

特性 LoRA QLoRA
基础模型精度 通常为16位（FP16/BF16） 4位量化（NF4/FP4）
内存占用 较高（需加载完整精度模型） 极低（模型内存减少70-80%）
适用硬件 需要专业级GPU 消费级GPU即可微调超大模型
性能保持 接近全参数微调 几乎无损（在多个基准测试中匹配16位性能）
代表项目 Alpaca-LoRA Guanaco系列模型（在Vicuna基准上击败GPT-4当时版本）

技术细节

1. 4位NormalFloat（NF4）数据类型

  • 专门为神经网络权重设计（通常服从正态分布）
  • 相比普通4位整数量化，精度更高

2. 双量化（Double Quantization）

  • 第一次量化：模型权重 → 4位
  • 第二次量化：第一次量化的常数因子 → 8位
  • 每参数额外节省约0.37位

3. 分页优化器（Paged Optimizers）

  • 当GPU显存不足时，自动将优化器状态转移到CPU RAM
  • 需要时再按页换回GPU，避免内存溢出错误

补充：Prompt Tuning（提示微调）

Prompt Tuning 是一种面向大型语言模型（LLM）的软提示微调方法（Soft Prompting / Parameter-Efficient Tuning），其核心思想是在不修改模型原有参数的前提下，通过学习一段可训练向量（soft prompt）附加到输入前端，从而让模型更好地适应特定任务。与传统的监督微调（SFT）或 RLHF 不同，Prompt Tuning 并不改动 Transformer 的权重，而是通过优化向量表示来引导模型行为。

其优势在于极大地减少了训练参数量和算力需求，使得在资源有限的情况下可以高效地定制模型任务能力；同时，soft prompt 可以与不同任务共享或复用，从而形成模块化、可插拔的微调策略。典型实现中，soft prompt 向量在嵌入层与输入 token 拼接，然后送入原模型计算输出，训练目标通常与下游任务的监督信号一致，例如分类标签、生成文本或序列标注。

Prompt Tuning 常用于单任务适配、小样本迁移、低成本定制 Agent 行为等场景，但由于其本质仅影响输入侧的指引，对模型多步推理、复杂任务规划或跨任务泛化能力的提升有限，因此在多工具 Agent 或复杂规划系统中，往往需要结合 Instruction Tuning 或 LoRA/QLoRA 等方法来强化模型能力。
