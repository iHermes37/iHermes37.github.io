所有权和借用

为什么需要所有权与借用

所有权系统能静态避免 C/C++ 常见灾难：

  • Use-after-free：由于引用不能比所有者活得久（生命周期规则），编译器自动禁止悬空访问。
  • Double free：只有一个所有者，无法重复释放。
  • Data race：不可变引用可并发共享，但 mut 引用必须独占，天生避免数据竞争。
  • 零成本抽象：所有验证在编译期进行，无运行时开销，是 Rust 面向系统的优势。

核心思想概览

Rust 的内存安全来自一个思想：每块内存只有一个“主人”（所有者），其他访问者必须是“借用者”，所有者死亡后内存立即自动释放，没有 GC，没有悬空指针。

  • 所有权解决的是”谁负责清理”的问题，确保每个值都有明确的责任人。
  • 借用解决的是”如何安全共享”的问题，允许访问数据而不承担责任。

这个思想在编译期验证，不会产生运行时开销。Rust 定义了三个核心规则：

  1. 每个值有且只有一个所有者（Owner）
  2. 一个值在任意时刻要么有多个不可变引用，要么只有一个可变引用（mut），不可混用
  3. 当所有者离开作用域，该值自动释放（drop）

这些规则保证：

  • 不会 use-after-free
  • 不会 double free
  • 不会数据竞争（data race）
  • 不需要 GC 也能达到 C/C++ 水平的性能

所有权

所有权就是内存的归属权。当一个值绑定到变量时，这个变量就是它的 唯一所有者。

深层价值

  • 静态内存安全模型（No GC，No UAF）：Rust 在类型系统层面解决 C/C++ 的大部分内存安全漏洞。适用于区块链虚拟机、共识算法、序列化引擎、数据库引擎、逆向分析工具等。
  • 显式资源管理（RAII 强化版）：不仅自动释放，还防止错误释放。这比 C++ RAII 更严格，因为 Rust 默认禁止复制资源。
  • 完美匹配高性能系统模型：没有 GC，没有暂停，零成本抽象

所有权的三条核心规则

规则一：每个值都有一个所有者（owner）

规则二：同一时间只能有一个所有者

规则三：当所有者离开作用域，值被丢弃（内存被释放）

借用（访问，但不拥有）

借用就是引用（reference）：允许你访问值但不拥有它。

引用类型

借用是 Rust 的第二条安全线。借用分两类：

  • 不可变借用：&T
  • 可变借用：&mut T

借用的核心约束是：在同一段时间内，要么有任意多个不可变引用，要么只能有一个可变引用。

这个思想本质是 编译期的数据竞争规避。

不可变借用：只读访问

不可变借用允许共享访问，因为只读不影响安全性。

可变借用：独占访问

可变借用保证：

  • 不会有两个线程同时写
  • 不会读写冲突

这让 Rust 的并发模型天生安全。

借用规则（编译时检查）

规则一：任意时刻，要么只能有一个可变引用，要么只能有多个不可变引用

规则二：引用必须总是有效的（不能出现悬垂引用）

场景 推荐方式 示例
只读访问 不可变借用 fn f(s: &str) -> usize
修改数据 可变借用 fn f(s: &mut String)
消费数据 获取所有权 fn f(s: String) -> String
返回新数据 转移所有权 fn create() -> Vec<i32>
共享访问 多个不可变引用 let r1 = &x; let r2 = &x;
独占修改 单个可变引用 let r = &mut x;
