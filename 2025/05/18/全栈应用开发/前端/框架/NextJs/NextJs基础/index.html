为什么要有动态路由？

动态路由是现代 Web 开发中非常重要的功能，它允许我们根据 URL 的不同部分动态加载不同的内容或执行不同的逻辑。动态路由的核心作用是 让 URL 更灵活、更语义化，并支持数据驱动的页面渲染。以下是动态路由的主要用途和优势：

1. 动态路由的核心作用

动态路由的核心是 根据 URL 的变化动态匹配不同的内容，而不是为每个可能的 URL 单独创建静态文件。例如：

  • /products/1 → 显示 ID 为 1 的产品
  • /blog/hello-world → 显示标题为 hello-world 的博客文章
  • /users/john → 显示用户名为 john 的用户信息

对比静态路由

  • 静态路由：每个页面需要单独创建文件（如 page1.html, page2.html），不适合大量动态内容。
  • 动态路由：通过 URL 参数（如 /products/:id）动态匹配内容，减少重复代码。

2. 动态路由的主要用途

(1) 数据驱动的页面渲染

动态路由允许我们根据 URL 的不同部分加载不同的数据，从而实现 单页应用（SPA） 或 服务端渲染（SSR） 的动态内容。

示例：电商网站的产品详情页

  • URL: /products/123
  • 后端逻辑：从数据库查询 id=123 的产品信息并渲染页面。
  • 前端逻辑（React/Vue）：通过 useParams() 或 useRouter() 获取 id=123 并加载数据。

(2) SEO 友好的 URL

动态路由可以让 URL 更语义化，提高搜索引擎优化（SEO）效果。

对比

  • ❌ 静态 URL（不友好）：/product?id=123
  • ✅ 动态 URL（友好）：/products/123 或 /products/nike-shoes

搜索引擎更喜欢结构化的 URL（如 /products/:id 或 /blog/:slug），因为它能更好地理解页面内容。

(3) 支持 RESTful API 风格

动态路由可以模拟 RESTful API 的路由结构，使前后端交互更直观。

示例

  • GET /users/1 → 获取用户 ID=1 的信息
  • POST /users → 创建新用户
  • PUT /users/1 → 更新用户 ID=1 的信息

(4) 减少重复代码

动态路由可以避免为相似的页面创建大量重复文件。

对比

  • ❌ 静态方式（重复代码）：
    • /about.html（关于我们）
    • /contact.html（联系我们）
    • /team.html（团队介绍）
  • ✅ 动态方式（减少重复）：
    • /pages/[page].tsx → 根据 page 参数动态加载内容（如 about、contact、team）。

(5) 支持嵌套路由

动态路由可以结合嵌套路由，实现更复杂的页面结构。

3. 动态路由的技术实现

动态路由通常通过 URL 参数 或 路径匹配 实现：

(1) URL 参数（Query Parameters）

  • 格式：?key=value（如 /search?q=react）
  • 适用场景：搜索、过滤等简单查询。

示例（React Router）

(2) 路径参数（Path Parameters）

  • 格式：/path/:param（如 /products/123）
  • 适用场景：动态内容加载（如产品详情页、用户主页）。

示例（Next.js 动态路由）

4. 动态路由的优势

优势 说明
灵活性 支持任意 URL 结构，适应不同业务需求
SEO 友好 结构化 URL 更容易被搜索引擎抓取
减少重复代码 通过参数动态加载内容，避免重复文件
RESTful 风格 模拟 API 路由，前后端交互更直观
嵌套路由支持 实现复杂的页面层级结构

5. 动态路由 vs 静态路由

对比项 动态路由 静态路由
URL 结构 /products/:id（灵活） /product1.html（固定）
数据加载 动态加载（如数据库查询） 静态内容（如 HTML 文件）
SEO 友好性 高（结构化 URL） 低（无参数）
维护成本 低（减少重复代码） 高（需手动管理多个文件）
适用场景 动态内容（如电商、博客） 静态内容（如公司介绍页）

6. 动态路由的典型应用场景

(1) 电商网站

  • /products/123 → 显示产品详情
  • /products/category/shoes → 显示分类商品

(2) 博客系统

  • /blog/hello-world → 显示文章
  • /blog/tech → 显示技术分类文章

(3) 用户管理

  • /users/1 → 显示用户信息
  • /users/1/orders → 显示用户订单

(4) 搜索功能

  • /search?q=react → 搜索 React 相关内容

7. 总结

动态路由的核心价值在于：

  1. 灵活性：支持任意 URL 结构，适应不同业务需求。
  2. SEO 友好：结构化 URL 更容易被搜索引擎抓取。
  3. 减少重复代码：通过参数动态加载内容，避免重复文件。
  4. RESTful 风格：模拟 API 路由，前后端交互更直观。
  5. 嵌套路由支持：实现复杂的页面层级结构。

动态路由是现代 Web 开发的核心技术之一，广泛应用于电商、博客、社交平台等需要动态内容的场景。

错误原因

在 Next.js 中，动态路由的路径名称（如 [slug]）必须唯一。如果你在同一个层级下定义了多个动态路由，但使用了不同的名称（如 [about] 和 [help]），Next.js 会报错：

示例错误配置

解决方案

1. 确保动态路径名称唯一

如果你需要多个动态路由，确保它们位于不同的目录层级，或者使用相同的 [slug] 名称（但逻辑不同）。

✅ 正确做法

  • 顶级 [slug].js 可以匹配 /product/1、/about 等。
  • blog/[slug].js 可以匹配 /blog/hello-world。

✅ 如果必须区分不同动态路由

  • about/[id].js 和 help/[id].js 是不同的动态路由，不会冲突。

2. 检查动态路由的命名

确保你的动态路由文件名遵循 Next.js 的规则：

  • 动态路径必须用 [] 包裹（如 [slug]、[id]）。
  • 同一层级不能有不同名称的动态路径（如 [about] 和 [help] 不能共存）。
