使用示例

关键点总结

  1. 特异性：当多条规则作用于同一元素时，CSS 会根据选择器的特异性来决定哪条规则生效。其权重通常计算为：内联样式(1000) > ID(100) > 类/伪类/属性(10) > 元素/伪元素(1) > 通用选择器(0)。
  2. 组合使用：选择器可以组合使用以实现更精确的目标，例如：ul.menu > li.active a。
  3. 伪类 vs 伪元素：
    • 伪类：用于选择处于特定状态的元素（如 :hover, :checked）。
    • 伪元素：用于选择元素的特定部分（如 ::before, ::first-line）。在 CSS3 中，伪元素使用双冒号 :: 以与伪类区分，但单冒号 : 对于 CSS1 和 CSS2 的伪元素仍然有效。

盒模型

CSS 盒模型是 CSS 布局的基础，它定义了 HTML 元素在页面上所占据的空间。理解盒模型对于创建复杂的网页布局至关重要。可以将每个 HTML 元素都想象成一个盒子，这个盒子包含内容，内边距，边框和外边距。

  • 内容区（Content Area）: 这是元素实际的内容所在，例如文本、图片等。可以使用 width 和 height 属性控制这个区域的大小。
  • 内边距（Padding）: 围绕内容区域的空白区域，位于内容区域和边框之间。可以使用 padding 属性来调整内边距的大小。padding 有四个方向的属性：padding-top、padding-right、padding-bottom 和 padding-left，也可以使用简写属性 padding 来设置所有方向的内边距。
  • 边框（Border）: 围绕内容和内边距的一条线。可以使用 border 属性设置边框的样式、宽度和颜色。与 padding 类似， border 也具有四个方向属性，可以使用 border-top、 border-right、 border-bottom、 border-left 来设置特定方向的边框，也可以使用 border 简写属性来设置。
  • 外边距（Margin）: 围绕元素边框的空白区域，用于控制元素与其他元素之间的距离。可以使用 margin 属性来调整外边距的大小。同样，margin 也具有四个方向属性： margin-top、margin-right、 margin-bottom 和 margin-left，并且可以使用简写属性 margin 来设置。

布局系统

流式布局【BFC】

BFC（Block Formatting Context，块级格式化上下文） 是 CSS 渲染过程中的一个独立布局区域。它规定了内部元素的布局规则，并且 与外部环境隔离，即 BFC 内的元素不会影响外部的布局，反之亦然。

BFC 的本质：创建一个独立的布局容器，内部规则不影响外部。

核心作用：

  1. 阻止外边距合并
  2. 清除浮动（解决高度塌陷）
  3. 隔离浮动元素覆盖

常用触发方式：overflow: hidden（最安全无副作用）。

弹性盒布局【Flexbox】

Flexbox 是 CSS3 提供的现代一维布局模型，专门解决传统布局（如浮动和定位）的痛点，适合组件级布局（如导航栏、卡片排列、表单等）。

Flexbox 的核心是弹性分配空间，通过容器和项目的属性组合，可以轻松实现：

  1. 对齐控制（水平/垂直居中）
  2. 动态尺寸（等分、填充剩余空间）
  3. 顺序调整（order）
  4. 响应式排列（flex-wrap）

网格布局【Grid】

Grid 是 CSS 提供的二维布局系统，比 Flexbox 更强大，适合构建复杂的页面结构（如仪表盘、杂志排版等）。

容器（Grid Container）与项目（Grid Items）

  • 容器：通过 display: grid 或 display: inline-grid 定义。
  • 项目：容器的直接子元素自动成为 Grid 项目。

网格线（Grid Lines）与轨道（Grid Tracks）

  • 网格线：划分网格的垂直线和水平线（从 1 开始编号）。
  • 轨道：相邻网格线之间的空间（行或列）。

弹性盒从内容出发。一个使用弹性盒的理想情形是你有一组元素，希望它们能平均地分布在容器中。你让内容的大小决定每个元素占据多少空间。如果元素换到了新的一行，它们会根据新行的可用空间决定它们自己的大小。

网格则从布局入手。当使用 CSS 网格时，你先创建网格，然后再把元素放入网格中，或者让自动放置规则根据把元素按照网格排列。我们能创建根据内容改变大小的网格轨道，但整个轨道的大小都会随之改变。

当你使用弹性盒，并发现自己禁用了一些弹性特性，那你可能需要的是 CSS 网格布局。例如，你给一个弹性元素设置百分比宽度来使它和上一行的元素对齐。这种情况下，网格很可能是一个更好的选择。

IFC

IFC 是 CSS 渲染过程中行内级元素（inline-level elements）的布局环境，决定了行内元素如何水平排列、对齐和换行。它是与 BFC（块级格式化上下文） 对应的概念，专注于处理行内内容的排版。

IFC 的触发条件

以下情况会创建一个 IFC：

  • 元素包含行内级内容（如文本、<span>、<a>、display: inline 或 inline-block 的元素）。
  • 容器未触发 BFC、FFC 或 GFC（即未设置 float、position: absolute、display: flex/grid 等）。

IFC 的核心特性

特性 说明
水平排列 行内元素从左到右（或从右到左）依次排列，直到占满行宽后换行。
对齐方式 通过 vertical-align 控制行内元素在行框（line box）内的垂直对齐。
行高（line-height） 决定行框的高度，影响文本的垂直间距。
空白符处理 默认合并连续空格和换行符（可通过 white-space 属性修改）。
无法设置宽高 纯行内元素（display: inline）的宽高由内容决定，无法直接设置。
  • BFC：隔离布局，专治浮动塌陷和外边距合并。
  • IFC：行内元素排版，解决文本对齐和间隙问题。
  • FFC：弹性伸缩，轻松实现一维自适应布局。
  • GFC：网格划分，二维布局的终极解决方案。

视觉样式

CSS 层叠、优先级与继承详解

层叠

层叠规则决定了当多个样式规则作用于同一元素时，哪个样式会被最终应用。其判断顺序如下：

  1. 来源重要性（从高到低）：
    • !important 声明的样式（慎用）
    • 开发者编写的样式（作者样式表）
    • 用户代理样式（浏览器默认样式）
  2. 选择器优先级（见第二部分）
  3. 代码顺序：
    • 后定义的样式会覆盖先定义的样式（同优先级时）。

优先级

优先级通过权重计算，决定哪个选择器的样式更优先。权重由四部分组成（从高到低）：

选择器类型 权重值（a, b, c, d）
内联样式（style=""） 1, 0, 0, 0
ID 选择器（#id） 0, 1, 0, 0
类/属性/伪类 0, 0, 1, 0
元素/伪元素 0, 0, 0, 1

比较规则：从左到右逐级比较，数值高的优先级高。

* 通配符和继承的样式：权重为 0,0,0,0。

继承

某些样式属性会自动从父元素传递到子元素，而某些属性不会。

1. 可继承的属性

  • 文本相关：font-family、color、line-height、text-align
  • 列表相关：list-style
  • 可见性：visibility

2. 不可继承的属性

  • 盒模型相关：width、height、margin、padding、border
  • 定位相关：position、top、left
  • 背景相关：background

3. 强制继承

浮动元素的特点

  • 脱离文档流：浮动元素会脱离正常流，后续元素会“无视”它（但文字内容会环绕）。
  • 宽度收缩：浮动元素的宽度默认由内容决定（除非显式设置 width）。
  • 高度塌陷：父容器无法自动包裹浮动子元素（需清除浮动）
