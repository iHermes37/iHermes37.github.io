类型

除了string、number 和 boolean三个在JS就有的类型，注意到TS相较于其他编程语言，在类型声明使用上，类似Golang，类似在变量名后进行注释解释。类型别名的使用类似C++是struct定义，同时在此基础上，Ts引入联合类型，其是由两种或多种其他类型组成的类型，表示可能是这些类型中的任何一种的值。 我们将这些类型中的每一种都称为联合的成员。

接口同java使用类似。同时注意其不少TS独有的属性：

匿名函数与函数声明有点不同。 当一个函数出现在 TypeScript 可以确定如何调用它的地方时，该函数的参数会自动被赋予类型。

  • TypeScript 也有一个特殊的类型，any，当你不希望某个特定的值导致类型检查错误时，你可以使用它。当一个值的类型为 any 时，您可以访问它的任何属性（这又将是 any 类型），像函数一样调用它，将它分配给（或从）任何类型的值。
  • 除了原语之外，您会遇到的最常见的类型是对象类型。 这指的是任何带有属性的 JavaScript 值，几乎是所有属性！ 要定义对象类型，我们只需列出其属性及其类型。对象类型还可以指定它们的部分或全部属性是可选的。 为此，请在属性名称后添加 ?
  • 类型断言,有时你会得到关于 TypeScript 无法知道的值类型的信息。TypeScript 只允许类型断言转换为更具体或更不具体的类型版本。 此规则可防止 “impossible” 强制，例如：const x = “hello” as number;
  • JavaScript 有两个原始值用于表示不存在或未初始化的值：null 和 undefined。TypeScript 有两个对应的同名类型。这些类型的行为取决于您是否启用了 strictNullChecks 选项。

类型缩小：

类型缩小是 TypeScript 的核心特性，用于在代码逻辑中将变量的类型从宽泛类型推导为更具体的类型，从而增强类型安全性和智能提示。(为了解决联合类型导致的无法预测的错误)

方法 适用场景 示例
typeof 基本类型（string/number/boolean） if (typeof x === "string")
instanceof 类实例判断 if (x instanceof Date)
in 对象属性判断 if ("key" in obj)
相等性比较 字面量或特定值比较 if (x === null)
自定义类型谓词 复杂类型验证 function isFish(x): x is Fish
可辨识联合 联合类型的区分 shape.kind === "circle"

其他类型：

类型 描述 典型使用场景 注意事项
void 表示函数没有返回值 无返回值的回调函数 () => void 与undefined不同，void是类型而非值
object 表示所有非原始值类型（包含数组、函数等） 需要约束非原始值时 const obj: object = {a: 1} 不要与空对象{}混淆，函数也属于object
unknown 表示未知类型，比any更安全 动态数据解析 JSON.parse()的返回类型 使用前必须进行类型检查或类型断言

高级类型

索引访问类型

作用：通过索引（键名）动态获取其他类型的属性类型。
​​语法​​：Type["key"] 或 Type[keyof Type]
​​特点​​：

  • 类似 JavaScript 的对象属性访问
  • 可用于类型查询和组合

示例

应用场景：

  • 从复杂类型中提取子类型
  • 与 keyof 结合实现动态类型操作

条件类型

作用：根据条件（类型关系）选择不同的类型分支。
​​语法​​：T extends U ? X : Y
​​特点​​：

  • 类似三元运算符的类型版
  • 常与 infer 关键字配合使用

示例

应用场景：

  • 类型工具库（如 ReturnType、Parameters）
  • 处理泛型约束和类型转换

映射类型

作用：基于现有类型批量生成新类型（类似 map 操作）。
​​语法​​：{ [K in KeyType]: ValueType }
​​特点​​：

  • 可配合 keyof 遍历键名
  • 支持添加修饰符（readonly、?）

示例

应用场景：

  • 内置工具类型（Partial、Readonly、Pick）
  • API 响应类型的动态生成

模板字面类型

作用：通过模板字符串拼接生成字符串字面类型。
​​语法​​：text${Type}
​​特点​​：

  • 支持字符串拼接、联合类型展开
  • 可结合 infer 进行模式匹配

示例

应用场景：

  • 路由路径解析
  • 国际化键名生成
  • CSS 属性值约束

同C++的函数重载相比：

特性 TypeScript 函数重载 C++ 函数重载
语言类型 静态类型检查（编译时类型擦除） 静态编译型语言（运行时保留）

泛型函数

TypeScript 的泛型函数在功能上最接近 C++ 的模板函数

特性 TypeScript 泛型函数 C++ 模板函数
目的 编写可处理多种类型的通用函数 编写类型无关的通用代码
语法标记 使用 <T> 声明类型参数 使用 template<typename T> 声明
类型推导 支持从实参自动推断类型 支持从实参自动推导类型（C++17 增强）
延迟绑定 编译时（TypeScript）或运行时（JavaScript） 编译时实例化

不同的地方在于：

方面 TypeScript C++
类型系统 编译时类型检查，运行时擦除 编译时生成具体函数实例
性能影响 无运行时开销（JavaScript 无泛型） 可能引发代码膨胀（每个类型实例化一份代码）
约束机制 通过 extends 约束泛型类型 通过 concepts（C++20）或 SFINAE
默认类型 支持（T = defaultType） 支持（C++11 起）
多态行为 依赖运行时动态类型（JavaScript） 纯静态多态（编译时决议）

Ts其他比较独特的地方在于：

函数类型表达式

作用：描述普通函数的参数类型和返回值类型。
​​语法​​：(参数: 类型) => 返回值类型
​​特点​​：

  • 最常用的函数类型定义方式
  • 适用于普通函数、箭头函数、回调函数等

调用签名

作用：在对象类型中定义方法的类型，支持附加属性描述。
​​语法​​：{ (参数: 类型): 返回值类型; 其他属性... }
​​特点​​：

  • 可以同时描述函数和它的静态属性
  • 常用于接口或对象类型中的方法定义

构造签名

作用：描述类的构造函数类型（即 new 调用的类型）。
​​语法​​：new (参数: 类型) => 返回值类型
​​特点​​：

  • 用于定义可被 new 调用的类型
  • 常用于工厂函数或类类型约束

readonly

字段可以以 readonly 修饰符作为前缀。 这可以防止对构造函数之外的字段进行赋值。

请注意，没有额外逻辑的由字段支持的 get/set 对在 JavaScript 中很少有用。 如果您不需要在 get/set 操作期间添加其他逻辑，则可以公开公共字段。

TypeScript 对访问器有一些特殊的推理规则：

  • 如果 get 存在但没有 set，则属性自动为 readonly
  • 如果不指定setter参数的类型，则从getter的返回类型推断
  • getter 和 setter 必须有相同的 成员可见性

从 TypeScript 4.3 开始，可以使用不同类型的访问器来获取和设置。
