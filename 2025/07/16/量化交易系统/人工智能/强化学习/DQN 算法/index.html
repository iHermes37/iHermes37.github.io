DQN 算法

架构介绍

强化学习的核心是建模决策问题，并通过与环境的交互来学习最佳决策方案

  • 智能体（Agent）：学习的主体，负责做出决策。

  • 环境（Environment）：智能体交互的外部系统，提供状态和反馈。

  • 状态（State, s）：环境在某一时刻的描述。

  • 动作（Action, a）：智能体在给定状态下可执行的操作。

  • 奖励（Reward, r）：环境对智能体动作的即时反馈（标量信号）。

  • 策略（Policy, π）：智能体的行为规则，即从状态到动作的映射（确定性或随机性）。

  • 价值函数（Value Function）：评估状态的长期价值，分为：

    • **状态价值函数 Vπ(s)**：在策略 π 下从状态 s开始的期望累积奖励。
    • **动作价值函数 Qπ(s,a)**：在策略 π下从状态 s执行动作 a后的期望累积奖励。

  • 环境模型（Model）（可选）：对环境动态（状态转移和奖励）的预测。

强化学习的目标是找到最优策略 π∗，最大化累积奖励。核心问题包括：

  • 探索 vs 利用：平衡尝试新动作（获取信息）与选择已知最优动作（最大化奖励）。
  • 信用分配：将长期结果归因于短期动作。
  • 延迟奖励：当前动作可能影响未来多步的奖励。

强化学习的核心是通过试错学习最优策略

经验回放

在一般的有监督学习中，假设训练数据是独立同分布的，我们每次训练神经网络的时候从训练数据中随机采样一个或若干个数据来进行梯度下降，随着学习的不断进行，每一个训练数据会被使用多次。在原来的 Q-learning 算法中，每一个数据只会用来更新一次值。为了更好地将 Q-learning 和深度神经网络结合，DQN 算法采用了经验回放（experience replay）方法，具体做法为维护一个回放缓冲区，将每次从环境中采样得到的四元组数据（状态、动作、奖励、下一状态）存储到回放缓冲区中，训练 Q 网络的时候再从回放缓冲区中随机采样若干数据来进行训练。这么做可以起到以下两个作用。

现在的问题就已经转换为需要一组训练集，它能够提供一批四元组（s, a, r, s’），其中s’为s执行a后的下一个状态。如果能有这样一个四元组，就能够用来训练DNN了，这就是我们要介绍的经验回放缓存。

为什么需要经验回放？

  • 打破数据相关性：直接按时间顺序训练（如最新样本总是一连串“跳跃”动作）会导致DNN“偏科”，只擅长处理近期状态。随机抽取旧样本训练，能让模型学到更全面的策略。
    类比：考试前不能只复习最后一章，而要从整个课本中随机抽题练习。
  • 重复利用数据：每个样本可以被多次使用（节省与环境交互的成本）。

想象在玩一个闯关游戏：每执行一个动作（比如“跳跃”），会记住：

  • 当前状态（s：看到障碍物）
  • 执行的动作（a：跳跃）
  • 得到的奖励（r：+10分，因为跳过障碍）
  • 下一个状态（s’：落地后的新画面）
  • 是否结束（done：False，因为还没通关）

这些信息构成一条转移样本（s, a, r, s’, done），存入一个“记忆库”（即Replay Buffer）。

前面提到我们需要一批四元组（s, a, r, s’）来进行训练，因此我们需要缓存一批这样的四元组到经验池中以供训练之用。由于每次执行一个动作后都能转移到下一个状态，并获得一个reward，因此我们每执行一次动作后都可以获得一个这样的四元组，也可以将这个四元组直接放入经验池中。

训练样本的来源

  • 经验回放缓存（Replay Buffer）：存储智能体与环境交互的转移样本 (s,a,r,s′,done)其中 done表示是否终止状态（如游戏结束）。
  • 采样方式：从缓存中随机抽取一批样本（打破数据相关性，提升稳定性）。

我们知道这种四元组之间是存在关联性的，因为状态的转移是连续的，如果直接按顺序取一批四元组作为训练集，那么是容易过拟合的，因为训练样本间不是独立的！为解决这个问题，我们可以简单地从经验池中随机抽取少量四元组作为一个batch，这样既保证了训练样本是独立同分布的，也使得每个batch样本量不大，能加快训练速度。

具体例子

假设Replay Buffer中存储了以下样本：

状态 (s) 动作 (a) 奖励 (r) 下一状态 (s’) 结束 (done)
前方有障碍物 跳跃 +10 障碍物消失 False
靠近悬崖 左移 -50 掉入深渊（终止） True
击败小怪 攻击 +20 新怪物出现 False

训练时，随机抽取一批（如2条）样本：

  1. 第一条（跳跃成功）：学习“遇到障碍物时跳跃有正收益”。
  2. 第二条（掉下悬崖）：学习“靠近悬崖时左移是危险的”。

这样，模型就能综合历史经验，避免局部最优（比如一味左移）。

目标网络

现在我们的选择哪个动作，是由DNN来做决定的，因此我们需要训练DNN以使其能达到令人满意的表现。这显然是一个监督学习的问题，那么训练集是什么，标签是什么，损失函数又是什么？

首先，我们DNN的输出值，自然是在给定状态的情况下，执行各action后能得到的Q值。然而事实上我们在很多情况下并不知道最优的Q值是什么，比如自动驾驶、围棋等情况，所以似乎我们没法给出标签。但是什么是不变的呢？Reward！

如何通过Reward训练Q网络？

核心问题:

  • 没有真实Q值标签：在自动驾驶或围棋中，我们无法预先知道“在某个状态下左转的精确Q值应该是多少”。
  • 唯一已知：执行动作后的即时奖励r（如“撞车：-1000”，“安全行驶：+1”）。

解决方案：贝尔曼方程

通过以下公式递归定义Q值：$Q(s,a)=r+γmax⁡a′Q(s′,a′)$ 即：当前Q值 = 即时奖励 + 未来可能的最大收益的折扣

这样，即使不知道真实的 Q∗，也可以通过不断迭代逼近它。

Target (r+γmax⁡Q)中，r是真实观察到的奖励，而 max⁡Q 是DNN对未来奖励的预测。通过最小化TD误差，DNN实际上是在学习：如何将当前的即时奖励 r和未来预测的奖励 γmax⁡Q组合起来，形成对动作总价值的准确估计。

对状态s，执行动作a，那么得到的reward是一定的，而且是不变的！因此需要考虑从reward下手，让预测Q值和真实Q值的比较问题转换成让模型实质上在拟合reward的问题。
