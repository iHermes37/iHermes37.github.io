unidbg运行原理解读

核心概念

AndroidEmulator

AndroidEmulator 是 Unidbg 提供的 Android 进程模拟器，它负责创建 ARM 架构的运行环境，支持 32/64 位。

Memory（虚拟内存管理）

Unidbg 通过 Memory 模块管理 so 库的加载和 Android 进程的内存映射。

DalvikVM（虚拟机）

DalvikVM 是 Unidbg 提供的 Java 层虚拟机，用于 JNI 交互。

DalvikModule Module（SO模块）

  • DalvikModule 负责加载 Java 层 DEX 代码，同时支持解析 JNI 调用。
  • Module 代表一个已加载的 so 文件，支持解析导出函数及调用。

简单使用

目标So库函数调用

符号表

Symbol 的本质：ELF文件（如Android的SO库）包含 .dynsym（动态符号表）和 .symtab（静态符号表），记录了所有导出/导入的函数和全局变量。每个符号包含：

  • 名称（如 Java_com_example_encrypt）
  • 类型（函数/数据/未定义等）
  • 绑定属性（全局/局部）
  • 在内存中的地址（或偏移量）

执行流程

AbstractARMEmulator

主任务 (MainTask) 最高 核心业务逻辑（如JNI调用）
线程任务 (ThreadTask) 中 模拟多线程环境
信号任务 (SignalTask) 最低 处理异步信号
