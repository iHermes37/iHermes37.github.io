邻居子系统

为什么需要邻居子系统？

网络通信遵循分层模型：

  • 网络层（第三层）：使用 IP 地址进行逻辑寻址，实现主机到主机的通信。
  • 数据链路层（第二层）：使用 MAC 地址进行物理寻址，实现同一链路上设备到设备的通信。

当一台主机需要向同一局域网内的另一台主机发送一个 IP 数据包时，它必须将数据包封装在一个帧 中。这个帧的头部需要填入目的地的 MAC 地址。

邻居子系统就是为了解决这个问题：给定一个下一跳的 IP 地址，如何找到其对应的 MAC 地址？

L3地址何时需要转换为L2地址？

在网络通信中，L3地址（如IP地址）必须转换为L2地址（如MAC地址）才能实现数据包在物理网络中的实际传输。以下是详细的场景分析：

需要转换的场景

(1) 同一局域网（LAN）内的通信

源主机和目标主机位于同一IP子网（如192.168.1.2和192.168.1.3）。

转换过程：

  • 源主机通过ARP（IPv4）或ND（IPv6）解析目标IP对应的MAC地址。
  • 封装数据包时，L3（IP头）的目标地址是IP，L2（以太网帧头）的目标地址是MAC。

示例： 主机A（192.168.1.2）向主机B（192.168.1.3）发送数据时，需先获取B的MAC地址。

(2) 跨子网通信中的最后一跳

数据包经过路由器转发后，到达目标主机所在的本地子网。

转换过程： 路由器在转发到最终子网时，需将目标IP转换为目标主机的MAC地址（通过ARP/ND）。

示例：
主机A（192.168.1.2）访问服务器C（10.0.0.5），路由器在10.0.0.0/24子网内需解析C的MAC地址。

无需转换的场景

(1) 点对点链路（如PPP、串行线）

点对点连接只有两个端点，无需L2地址标识目标（链路层协议如PPP不依赖MAC地址）。

数据封装： 直接使用L3协议（如IP）封装，无需L2头部的源/目标地址字段。

示例： 两台路由器通过串行线（Serial）连接，IP包直接通过HDLC或PPP协议传输。

(2) 广播/组播通信
  • 广播（如ARP请求）： L2目标地址为FF:FF:FF:FF:FF:FF（无需解析特定MAC）。
  • 组播（如IPv6 ND）： L2目标地址为预定义的组播MAC（如33:33:00:00:00:01对应IPv6的ff02::1）。

触发条件

场景 是否需要L3→L2转换？ 依赖协议
同一子网的单播通信 是 ARP（IPv4）/ND（IPv6）
跨子网通信的最后一跳 是 ARP/ND
点对点链路（PPP/HDLC） 否 无
广播/组播通信 否（使用固定L2地址） 无
  • L3地址是逻辑寻址（全局唯一），L2地址是物理寻址（本地有效）。
  • 在多访问网络（如以太网、Wi-Fi）中，必须通过L2地址实现精准投递。
  • 点对点链路和广播/组播通信是例外，无需动态地址解析。
  • 现代交换网络通过MAC表优化了L2流量的转发效率，但ARP/ND仍是基础机制。

概念

邻居子系统是 Linux 网络协议栈中一个至关重要的组件，它负责管理与直接相连的邻居节点（在同一本地链路/广播域上的设备）的通信。它的核心功能是将网络层地址（如 IPv4/IPv6 地址）解析为数据链路层地址（如 MAC 地址），并维护这些映射关系的状态。

基础结构

邻居子系统的关键组成部分：

组件 描述
邻居表 核心数据库，存储了所有已知的邻居信息。可以通过 ip neighbor show 或 arp -a 命令查看。
状态机 定义了每个邻居表项可能处于的状态，这是邻居子系统最精妙的设计之一。
协议引擎 实现了具体的地址解析协议，如 arp_solicit (用于IPv4 ARP) 和 ndisc_solicit (用于IPv6 ND)。
输出队列 当需要解析地址时，目的地为该邻居的数据包会被暂时存放在一个队列中，等待解析完成后再发送。
垃圾回收机制 定期清理过时、无效的邻居表项，以释放资源。

邻居的定义

在计算机网络中，邻居（Neighbor） 是指与你的主机直接相连且无需经过路由器转发即可通信的其他设备。具体条件包括：

  • 同一局域网（LAN）：通过共享介质（如以太网交换机）或点对点链路直接连接。如果两台主机被L2上的某个设备（如网桥）隔离，它们仍然是邻居。
  • 相同的L3网络配置：在IP网络中，两台主机至少有一个接口属于同一IP子网（如192.168.1.2/24和192.168.1.3/24）。从路由表看，目标主机仅有一个L3跃点（即无需网关转发）。

举例来看：主机A（192.168.1.2）和主机B（192.168.1.3）连接同一交换机。

  • 邻居关系：是（同一子网，L2直接可达）。
  • 通信流程：
    1. A通过ARP查询B的MAC地址。
    2. A封装以太网帧（目标MAC=B的MAC）并发送。
    3. 交换机根据MAC表将帧转发给B。

邻居的通信特性

类型 共享介质（如传统以太网） 点对点链路（如串行线）
连接方式 多主机共享同一介质（如集线器） 仅两个端点直接连接
寻址需求 需要MAC地址（广播/单播） 无需链路层地址（仅两端通信）
冲突处理 需要CSMA/CD检测冲突 无冲突（全双工）或独占介质
典型场景 旧式同轴电缆以太网、Wi-Fi PPP协议、光纤直连

邻居协议的作用

  • 地址解析：
    • IPv4：通过ARP将IP地址解析为MAC地址。
    • IPv6：通过ND（Neighbor Discovery） 实现类似功能。
  • 可达性检测：定期验证邻居是否在线（如ARP保活、ND的NUD机制）。

现代以太网中的邻居

交换机取代共享介质：现代LAN通常使用交换机（Switch），每个主机通过独立UTP线缆连接交换机端口：

  • 全双工模式：主机和交换机可同时收发数据（无需CSMA/CD）。
  • 自动协商：双工模式和速率（如100Mbps/1Gbps）由设备自动协商。

邻居关系不变：即使通过交换机连接，同一子网的主机仍视为邻居（因L2直接可达）。

邻居与非邻居的对比

特征 邻居（同一LAN） 非邻居（跨子网）
通信方式 直接L2转发（MAC地址） 需路由器L3转发（IP路由）
协议依赖 ARP（IPv4）/ND（IPv6） 路由协议（如OSPF、BGP）
示例 办公室内两台电脑通过交换机相连 公司总部与分支机构通过VPN通信

路由

Linux网络中的路由是决定数据包从源主机到目的主机传输路径的过程，它基于内核维护的路由表进行转发决策

简单地讲，路由器就是一台网络设备，它配备多个网络接口卡、能利用它的网络知识正确转发人口流量

【路由器是具备多个网络接口（NIC）的专用设备，核心功能是基于IP地址（L3）在不同网络间转发数据包。】。

其可以运行路由协议（如OSPF、BGP）动态学习网络拓扑。并维护路由表（FIB），决定数据包的下一跳路径。

路由缓存

路由缓存则是为了优化路由查询性能而设计的快速查找结构。 当主机需要发送数据包时，内核首先检查目的IP地址，若不属于本地网络则查询路由表确定下一跳网关接口；

路由表包含目标网络、子网掩码、网关和出接口等关键信息，可通过ip route show查看。

路由缓存作为性能优化层，保存最近使用过的路由查询结果，采用哈希表实现快速O(1)查找，避免每次数据包发送都遍历庞大的路由表；

缓存条目包含源/目的IP、TOS等完整流信息，支持精确匹配且具备过期淘汰机制。在Linux 2.6.23之前路由缓存独立存在，此后被更高效的流匹配算法替代并最终整合进协议栈底层。

现代Linux系统通过FIB（转发信息库） TRIE算法优化路由表查找，结合每CPU缓存结构和快速路径转发，在维持路由功能正确性的同时显著提升转发性能，使路由子系统能够支撑从嵌入式设备到核心路由器等各种场景的网络需求。

路由表

大多数主机通常只有一个网络接口，因而不能作为路由器。主机被配置为使用一个默认网关来访问外部地址。在上图中，目的地为子网10.0.0.0/24之外任何主机的流量（用0.0.0.0/0指定）都被送给网关10.0.0.1。对目的地为10.0.0.0/24网络内的主机的流量，则使用邻居子系统。

不考虑网络内主机扮演的角色，每个主机维护一张路由表，当需要处理网络流量的发送和接收时查询该路由表。路由器可能需要运行一般主机通常不会使用的特殊软件，称为路由协议。毕竟，路由器需要更多关于如何到达远程网络的知识，非路由器主机就是依赖这些知识来访问远程网络的。

路由表无非就是许多路由的一个集合。一条路由就是一组参数，这些参数存储了往一个给定目的地转发流量所需要的信息。

路由表存储如何到达目标网络的规则，每条路由包含：

  • 目标网络（如10.0.0.0/24）。
  • 出口接口（如eth0）。
  • 下一跳地址（如10.0.0.1）。
  • 优先级/度量值（用于多路径选择）。

查找

Linux内核中的路由查找是数据包转发过程中最关键的环节之一，它通过查询路由表来确定数据包的下一跳地址和输出网络接口。

当网络层收到需要转发的数据包时，首先提取目的IP地址，然后按照一系列规则进行匹配：

优先检查是否有精确的主机路由，接着匹配网络路由，最后尝试默认路由，整个过程遵循最长前缀匹配原则。

Linux通过基于策略的路由机制支持多路由表架构，包括local、main、default等表，通过路由策略数据库（RPDB）的规则决定查询顺序，这种设计允许根据源地址、服务类型或其他数据包属性选择不同的路由路径。

现代Linux内核采用高度优化的FIB（Forwarding Information Base）TRIE查找算法，将路由表组织成二叉树结构实现高效搜索，同时结合每CPU缓存来存储最近使用的路由结果，避免对同一数据流的重复查找。对于本地回送地址、本机IP和直接相连的网络段，路由查找会快速识别并交由相应处理流程，而转发数据包则需确定下一跳网关和出口设备，整个查找过程深度融合Netfilter钩子点，为数据包过滤和修改提供切入点，最终生成的路由决策包含输出接口、下一跳网关、服务类型等完整转发信息，确保数据包能够正确抵达目标网络。
